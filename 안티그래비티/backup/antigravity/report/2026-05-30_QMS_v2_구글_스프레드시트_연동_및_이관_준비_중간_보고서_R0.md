## 2. 이상발생, 변수, 디버깅 내용 정리

### 2-1. 이상발생: 스프레드시트 2026년 데이터 유실 현상
* **증상:** 스프레드시트 연동 직후, 2026년도 자료가 그리드 및 차트에 오직 6건만 노출되는 심각한 유실 징후 감지.
* **기술적 원인:** 수천 건에 달하는 구글 시트 원장 데이터를 DB에 일괄 적재(Upsert)하는 도중, **중복 키(Duplicate Key) 충돌 에러(PGRST21000)**가 발생하여 중간에 동기화 트랜잭션이 중단 및 차단되고 있었음을 규명함.

---

### 2-2. 취약점 분석 및 고유 ID 해시 유실 디버깅
* **분석:** 고유 식별자(`id`) 생성을 위해 `inspectionReportNo` 및 날짜 등을 조합한 뒤 base64로 해싱한 문자열을 30자로 자르는 `.substring(0, 30)` 로직을 적용 중이었습니다.
* **발견된 변수:** 해싱된 문자열의 끝부분에 순차 인덱스(`_index`)를 붙인 뒤 30자로 자르게 됨에 따라, 고유성을 보장해 주어야 할 핵심 index 파트가 30자 한계선 뒤로 밀려나 통째로 잘려 나갔습니다. 이로 인해 동일한 base64 30자 문자열이 대량 생성되어 고유 인덱스 충돌을 유발했습니다.
* **디버깅 및 해결 코드:** 해싱된 문자열 자체를 20자로 먼저 여유 있게 자른 뒤, 그 뒤에 고유 인덱스 접미사를 명시적으로 결합하는 안전한 구조로 핫픽스를 배포하여 충돌을 100% 해결하였습니다.
  ```javascript
  // 핫픽스 개선 코드
  const hashPart = base64Hash.substring(0, 20);
  const finalUniqueId = `${hashPart}_${index}`; // 100% 고유성 영구 사수 완료
  ```
  이 핫픽스를 프론트엔드(`InboundHistory.jsx`)와 백엔드 API(`api/sync-sheets.js`) 양쪽에 동일한 정합성으로 긴급 배포 완료함으로써, **총 7,477건**에 달하는 대용량 데이터의 무결 동기화를 완벽히 성취해 내었습니다.
