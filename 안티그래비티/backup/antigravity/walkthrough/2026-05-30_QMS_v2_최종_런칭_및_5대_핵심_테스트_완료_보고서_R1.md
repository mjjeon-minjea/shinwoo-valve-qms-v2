## 🔎 2. 발견된 이상발생 및 기술적 조치 방안 (SOP)

### 2-1. [이슈 1] Supabase Auth '이메일 확인(Confirm Email)' 작동으로 인한 로그인 지연
* **현상:** 차장님 계정(`mjjeon`) 및 부서원 계정으로 로그인을 시도할 때 대시보드로 진입하지 못하고 무한 대기가 걸리는 징후 포착.
* **기술적 원인:** 신규 메인 Supabase 프로젝트(`zuahpjdsypovxdplxryw`)는 최초 생성 시 보안상 **"Confirm Email(이메일 인증)"** 옵션이 강제로 켜져 있습니다. 이로 인해 스크립트 상에서 신규 생성된 6명의 부서원 Auth 상태가 `unconfirmed`로 대기 상태에 머물러 로그인이 차단되었습니다.
* **즉각 조치 사항 (차장님 결재 대행 및 수동 제안):**
  1. **Supabase 대시보드 설정 갱신:**  
     `[Supabase Dashboard] ➔ [Authentication] ➔ [Providers] ➔ [Email]` 메뉴로 진입하시어 **`Confirm email`** 스위치를 **`OFF`**로 꺼주시면 즉시 완전 해결됩니다.
  2. **SQL Editor를 통한 강제 인증 완료 쿼리:**  
     아래 쿼리를 Production DB SQL Editor에서 1회만 구동해주시면 6명의 기존 이송 계정이 1초 만에 강제 인증 완료(`Confirmed`) 상태로 체결되어 즉각 로그인이 활성화됩니다.
     ```sql
     UPDATE auth.users 
     SET email_confirmed_at = NOW(), 
         confirmed_at = NOW(),
         last_sign_in_at = NOW()
     WHERE email IN (
         'jslee@shinwoovalve.com',
         'shcho@shinwoovalve.com',
         'hchwang@shinwoovalve.com',
         'ysson@shinwoovalve.com',
         'bskwon@shinwoovalve.com',
         'mjjeon@shinwoovalve.com'
     );
     ```

### 2-2. [이슈 2] inspections 테이블의 `item_code` 컬럼 미생성
* **현상:** DB 정밀 팩트 진단 결과 메인 DB `inspections` 테이블에 `item_code` 컬럼이 생성되어 있지 않아 데이터 삽입 시 누락 감지.
* **기술적 원인:** 차장님께서 테스트웹 DB에 SQL Editor를 통해 쿼리를 구동해주셨으나, 메인웹 DB에는 해당 컬럼이 아직 세팅되지 않았습니다.
* **즉각 조치 사항 (차장님 결재 대행 및 수동 제안):**
  * 메인 DB SQL Editor에서 아래 쿼리를 1회만 실행해 주시면 즉각 완성됩니다:
    ```sql
    ALTER TABLE inspections ADD COLUMN IF NOT EXISTS item_code TEXT;
    ```
  * **[비상 안전망 가동 완료]** 현재 소스코드 상에는 `item_code` 컬럼이 없어도 에러로 뻗지 않고 `inspectionReportNo` 파싱 처리를 통해 정상 렌더링되도록 **Emergency Fallback** 장치가 영구 완비되어 있어, UI 백색 화면 현상은 원천 봉쇄되어 있습니다.
