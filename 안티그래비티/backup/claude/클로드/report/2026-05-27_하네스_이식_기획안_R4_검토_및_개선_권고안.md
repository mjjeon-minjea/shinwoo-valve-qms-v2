## 3. 잔존 문제점

### 🔴 지시 불이행 — CLAUDE.md가 여전히 존재한다

차장님의 지시: **"CLAUDE.md 안된다니깐 무조건 GEMINI.md에 맞춰"**

R4 변경 영향 범위 (1절):
```
CLAUDE.md [NEW] - 하네스 트리거 포인터 및 변경 이력 수록  ← 여전히 존재
.agent/rules/GEMINI.md (혹은 전역 GEMINI.md) [MODIFY]    ← 추가됨
```

R4 1단계 구현 사양에도 `CLAUDE.md` 신규 생성 및 템플릿이 그대로 남아 있다. GEMINI.md를 함께 수정하는 것으로 보완했지만, CLAUDE.md를 같이 유지하는 것은 차장님 지시와 정면으로 배치된다.

**안티그래비티(Gemini)는 CLAUDE.md를 읽지 못한다.** CLAUDE.md를 만들어도 하네스 규칙이 Gemini에게 전달되지 않는다. 따라서 CLAUDE.md는 기획안에서 완전히 제거하고, 동일한 내용을 전역 `C:\Users\mjjeon\.gemini\GEMINI.md`에 이식하는 것으로 대체해야 한다.

**R5 반영 지침:**
- 변경 영향 범위에서 `CLAUDE.md [NEW]` 항목 삭제
- 1단계 구현 사양에서 CLAUDE.md 생성 및 템플릿 전부 삭제
- 1단계 롤백 시나리오에서 `git checkout -- AGENTS.md CLAUDE.md` → `git checkout -- AGENTS.md` 로 수정
- 1단계 검증 기준에서 `CLAUDE.md` 언급 제거
- GEMINI.md가 CLAUDE.md의 역할을 완전히 대체함을 명시

### 🟠 기술 결함 — R4의 RegEx가 여전히 단일 문자를 캡처할 수 있다

R4의 RegEx: `api\.fetch\(\s*['"]\/([^/'"]+?)[/'"]?`

문제는 **lazy quantifier(`+?`)와 optional 종료자(`[/'"]?`)의 조합**이다. JavaScript NFA 정규식 엔진에서 lazy 수량자는 패턴 전체가 성립하는 최소 길이만 캡처한다. `[/'"]?`는 0회 매칭이 가능하므로, `([^/'"]+?)`는 단 1글자만 캡처하고 패턴이 성립해버린다.

실제 테스트:
```js
const re = /api\.fetch\(\s*['"]\/([^/'"]+?)[/'"]?/g;
'api.fetch(\'/inspections/batch\')'.match(re);
// 캡처 그룹 결과: 'i'  ← 틀림
```

**올바른 RegEx (R3 검토에서 제시한 것):**
```js
// greedy, no lazy, no optional suffix
/api\.fetch\(\s*['"]\/([^/'"]+)/g

// 테스트
'api.fetch(\'/inspections/batch\')' → 캡처: 'inspections'  ✅
'api.fetch(\'/settings/global\')'   → 캡처: 'settings'     ✅
'api.fetch(\'/weekly_reports\')'    → 캡처: 'weekly_reports' ✅
```

`[^/'"]+` (greedy)는 `/`, `'`, `"` 를 만나기 전까지 최대한 많이 캡처한다. `/inspections/batch`에서 `inspections`까지 잡고 `/`에서 멈추므로 정확하다. lazy나 optional suffix가 필요 없다.

### 🟡 경미 — GEMINI.md 수정 대상이 모호하다

변경 영향 범위에 `.agent/rules/GEMINI.md` **(혹은 전역 `GEMINI.md`)**로 기재되어 "혹은"이 들어가 있다. 어느 파일을 실제로 수정할지 명확하지 않다. 차장님 지시에 따르면 전역 `C:\Users\mjjeon\.gemini\GEMINI.md`가 수정 대상이다.

**수정:** "혹은" 표현 삭제, `전역 C:\Users\mjjeon\.gemini\GEMINI.md [MODIFY]`로 단일화.
