# GEMINI.md 규칙 연동 및 통합 초안 리포트 (Plan)

## 1. 개요 및 목적
현재 에이전트(안티그래비티)가 시작 시 기본으로 읽어들이는 전역 컨텍스트는 `GEMINI.md` 하나뿐이므로, 기존에 별도로 분리되어 있던 3개의 주요 규칙 파일(`.agent/rules/01_tech_stack.md`, `02_dnas_process.md`, `03_archiving.md`)의 내용이 자동으로 연동되지 않는 문제가 있었습니다.
이를 해결하기 위해 `GEMINI.md` 내부에 세 파일의 핵심 조항을 물리적으로 완전히 통합(연동)시키고, 조항 번호를 체계적으로 재정렬(넘버링)하며, 각 조항이 기원한 정확한 파일 경로를 명기하여 완벽한 '마스터 하네스 인덱스'로 기능하도록 수정합니다.

## 2. 전후(Before vs After) 대비표

| 구분 | 수정 전 (Before) | 수정 후 (After) |
|---|---|---|
| **규칙 연동 방식** | `GEMINI.md`에는 01~03번 파일의 **경로만 단순 텍스트로 나열**되어 있어, AI가 해당 내용을 지시 없이 스스로 읽지 못함 | 세 파일의 **모든 본문 내용이 `GEMINI.md` 안으로 100% 흡수(연동) 삽입**되어 AI 기동 즉시 전역 룰로 즉각 활성화됨 |
| **조항 넘버링** | 파일마다 1, 2, 3으로 중구난방 표기됨 | 제1장(Rules), 제2장(Skills) 등 대분류 산하에 **[조항 1.1] ~ [조항 1.10]** 형식으로 전체 규칙을 일련번호로 재정렬 |
| **출처 경로 명시** | 없음 | 각 조항 제목 옆에 `(출처 경로: .agent/rules/...)` 형식으로 원본 물리 파일 경로를 명시하여 무결성 확보 |
| **배포 잠금 조항** | `GEMINI.md`와 `02_dnas_process.md`에 파편화되어 존재함 | 제3장 깃허브 배포 파이프라인 섹션에 **[조항 3.1] 다단계 승인 배포 프로토콜(Hard Lock 포함)**로 완전 통합 |
