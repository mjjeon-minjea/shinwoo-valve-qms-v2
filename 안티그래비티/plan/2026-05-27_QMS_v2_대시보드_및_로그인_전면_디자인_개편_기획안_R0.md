# QMS v2 대시보드 및 로그인 전면 디자인 개편 기획안 (R0)

본 기획안은 신우밸브주식회사 품질보증부 전민재 차장님의 명확하고 훌륭한 3대 디자인 의사결정 지침에 따라, QMS v2 프론트엔드의 비주얼과 사용성을 프리미엄 수준으로 끌어올리기 위한 **로그인 및 대시보드 전면 리디자인 1단계 기획안**입니다.

---

## 🧭 1. 개편의 핵심 디자인 가이드라인

차장님의 확정 사항에 맞춰 **"다크그레이 사이드바 & 프리미엄 라이트 본문"**의 강력한 비주얼 대비 사상을 수립합니다.

```
┌────────────────────────────────────────────────────────┐
│  [다크그레이 사이드바]  │  [프리미엄 라이트 본문]           │
│  - Slate #1e293b       │  - 백그라운드: Slate-50/50        │
│  - HSL 222 47% 11%     │  - 헤더: Glassmorphism White      │
│  - 프리미엄 액티브 바   │  - 카드 테두리: Subtle Shadow     │
└────────────────────────────────────────────────────────┘
```

1. **좌측 사이드바 (배너):** 세련되고 깊이감 있는 **다크그레이(Dark Grey / Slate HSL) 색상**을 적용하여 묵직하고 전문적인 제어반 느낌을 확보합니다.
2. **나머지 전체 영역 (헤더 & 본문):** 은은한 화이트, 실버, 소프트 그레이를 믹싱한 **프리미엄 라이트 테마**를 기조로 하여 정보를 맑고 투명하게 표현합니다.
3. **전면 개편 범위:** 사용자 첫 접점인 **로그인 화면(`Hero.jsx`)**부터 메인 대시보드 및 통계 분석 컴포넌트 전체를 SB Admin 어드민 정석 디자인으로 개조합니다.
4. **다단계 안전 배포 원칙:** 로컬 주소에서 완벽히 수정한 후 테스트하고, 차장님 승인 하에 **테스트웹(`test-origin`)에 우선 반영**하여 검증을 완료한 뒤 실서버 배포는 사후 진행합니다.

---

## 🛠️ 2. 컴포넌트별 리디자인 세부 계획

### ① 로그인 화면 (`src/components/Hero.jsx`) [MODIFY]
* **기존:** 단순 백그라운드 블러와 파란색 계열의 투박한 박스 형태.
* **개편안:** 
  - 좌측에는 신우밸브의 40년 역사와 품질 슬로건을 담은 세련된 일러스트레이션 카드 레이아웃 배치.
  - 우측 로그인 폼 카드는 테두리 섀도우를 깊게 준 미니멀리즘 슬레이트 라이트 카드로 리디자인.
  - Supabase 연동 보안 패스워드 변경 모달 역시 모던 HSL 다크그레이와 소프트 레드를 조화롭게 믹싱하여 신뢰감 부여.

### ② 메인 레이아웃 및 사이드바 (`src/components/Dashboard.jsx` 등) [MODIFY]
* **기존:** 일반적인 플랫한 색상 매칭.
* **개편안:** 
  - 사이드바 백그라운드를 다크그레이(`bg-slate-900` / `#0f172a` 수준)로 변경하고, 선택된 활성 메뉴의 좌측에 4px 두께의 선명한 블루 인디케이터 바를 장착합니다.
  - 상단 헤더(`Header.jsx`)는 본문의 프리미엄 라이트 톤과 부드럽게 매칭되도록 투명 블러(`backdrop-blur-md bg-white/80`) 처리를 적용합니다.

### ③ 품질 대시보드 컴포넌트군 (전체 24개 UI 파일 대상 순차 리디자인)
* **대상 파일:** `Dashboard.jsx`, `WeeklyStatus.jsx`, `ProcessInspectionDashboard.jsx` 등
* **개편안:**
  - SB Admin 고유의 4대 품질 지표 카드(Primary, Success, Warning, Danger)에 그라데이션 및 은은한 박스 섀도우 장착.
  - Recharts 등 내부 차트 영역 배경을 무결한 화이트로 두고, 경계선을 미세 조정하여 비주얼 밸런스 확립.

## 🧪 3. 통합 검증 및 다단계 배포 프로세스

### 검증 절차
1. **로컬 개발서버 구동:** 로컬 환경에서 로그인 및 대시보드 전 영역 디자인 구현.
2. **1차 자가진단:** boundaries 아키텍처 제약 위반 에러 0건 및 린트 정상 통과 실측 검증.
3. **테스트웹 반영:** 차장님 결재 후 `git push test-origin main`을 실행하여 테스트 도메인([shinwoo-valve-qms.vercel.app](https://shinwoo-valve-qms.vercel.app))에 1차 실 환경 배포.
4. **차장님 최종 컨펌:** Vercel 환경에서 완벽한 비주얼이 이상 없이 표출되는지 크로스체크 후 승인 획득.
