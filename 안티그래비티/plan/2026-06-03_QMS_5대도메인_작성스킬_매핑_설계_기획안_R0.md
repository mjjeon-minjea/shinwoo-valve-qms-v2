## 1. 아키텍처 로직 구성 변경안 (AS-IS ➔ TO-BE)

현재 QMS 규칙 시스템은 문서의 아카이빙(저장, 리비전, 동기화)만 스킬로 위임하고 있으며, 각 문서의 실질적인 작성(Authoring) 단계는 지정된 스킬이 없이 에이전트의 임의 포맷에 의존하고 있습니다. 본 기획안은 .agent/skills에 내장된 기존 범용 스킬들을 QMS 5대 핵심 도메인에 명시적으로 매핑하고, QMS 제약사항(한글 작성, 저장경로 강제 오버라이드 등)을 각 스킬 및 규칙 파일에 강력하게 바인딩하는 것을 목표로 합니다.

```
[AS-IS: 작성 스킬 미지정 및 임의 형식 작성]
- DNAS 결재 문서 작성 시, 어떤 스킬을 사용하여 어떤 포맷으로 작성해야 하는지 강제 규정이 없음.
- superpower 계열 범용 스킬들이 기본 저장경로(docs/superpowers/plans/ 등)와 영문 작성 전제를 가지고 있어 QMS에 그대로 사용 시 오작동 및 경로 혼선 발생.

[TO-BE: 도메인별 작성 스킬 매핑 및 스킬 본문 보정]
- 규칙(02_dnas_process.md)에 도메인별 작성 스킬, 아카이빙 스킬, 자가검토 게이트 간의 표준 연계 순서 명문화.
- 4대 범용 스킬 정의서(SKILL.md) 내부에 "QMS 프로젝트 오버라이드 제약 사항"을 명시적으로 수술식 이식하여 스킬 동작 시 QMS 전용 경로 및 한글 작성을 완벽히 강제함.

   DNAS 문서 작성 요구
          │
          ├──➔ Plan (기획안)      ➔ writing-plans 스킬      ➔ [오버라이드] 안티그래비티\plan\ (한글)
          ├──➔ Task (작업 명세)    ➔ planning-with-files 스킬➔ [오버라이드] 안티그래비티\task\ (한글)
          ├──➔ Walkthrough (완료) ➔ internal-comms 스킬     ➔ [오버라이드] 안티그래비티\walkthrough\ (한글)
          └──➔ Report (특별 보고)  ➔ doc-coauthoring 스킬    ➔ [오버라이드] 안티그래비티\report\ (한글)
```
