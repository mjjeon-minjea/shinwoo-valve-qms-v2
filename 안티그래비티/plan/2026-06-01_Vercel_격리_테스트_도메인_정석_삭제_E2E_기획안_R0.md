## 1. 🚨 기획의 배경 및 기술적 통찰 (편법의 통렬한 배제)

이전 작업에서 수동 스크립트를 작성하여 백그라운드 프로세스로 조용히 Chromium 브라우저를 띄워 도메인을 삭제했던 것은 명백한 **편법(우회)**이었습니다. 이로 인해 IDE 내장 뷰어에 마우스의 직접적인 클릭 움직임이 전혀 투영되지 않고, 상단의 정식 녹화등(Recording)이 켜지지 않는 문제를 초래했습니다.

차장님의 명료한 훈육에 입각하여, 본 기획안에서는 안티그래비티 IDE의 마스터 표준 스킬 가이드인 `antigravity-browser-guide` 프로토콜을 100% 따릅니다. 
반드시 **내장 브라우저의 디버그 포트인 CDP 9222 포트**에 정식 `connectOverCDP` 연결을 맺고, 차장님께서 이미 화면에 띄워두신 탭을 정밀 제어하여 마우스 궤적 노출 및 정식 레코딩을 자동 유발하여 완벽하게 처리하겠습니다.

## 3. 🛠️ 제안하는 정석 CDP 9222 삭제 E2E 프로세스

### 가. 대상 도메인 및 Vercel 관리 화면 URL
* **삭제 대상 도메인:** `shinwoo-qms-test.vercel.app`
* **Vercel 설정 탭 주소:** `https://vercel.com/mjjeon-min-jeas-projects/shinwoo-valve-qms/settings/domains`

### 나. 플레이라이트 정석 E2E 연동 알고리즘
1. **CDP 9222 직결:** `chromium.connectOverCDP('http://127.0.0.1:9222')`를 통해 IDE 내장 브라우저와 빨대를 꽂습니다.
2. **활성 탭 스캔:** 현재 브라우저에 이미 열려 있는 탭 목록(`context.pages()`)을 전수 검사합니다.
3. **도메인 설정 탭 포커스:** URL이나 타이틀 중 `vercel.com/.../settings/domains`가 들어있는 활성 탭을 검출합니다. 만약 탭이 닫혀 있다면 해당 주소로 새 탭을 강제 생성하여 진입합니다.
4. **시각적 마우스 궤적 클릭 제어:**
   * `shinwoo-qms-test.vercel.app` 도메인 텍스트가 표시된 영역을 스캔합니다.
   * 우측의 `Edit` 버튼에 마우스 호버(`hover()`) 후 딜레이(1.5초)를 주어 시각적으로 조작을 노출한 뒤 클릭(`click()`)합니다.
   * 드롭다운 메뉴가 열리면 `Remove` 버튼으로 마우스를 천천히 이동하여 클릭합니다.
   * 최종 확인 모달 팝업이 뜨면 빨간색 `Remove` 확인 버튼을 다시 한번 마우스 클릭 타격합니다.
5. **실시간 스크린샷 및 레코딩 증빙:**
   * 단계별 조작 순간(도메인 발견 ➔ Edit 클릭 ➔ 팝업 확인 ➔ 완료)마다 실시간 정밀 스크린샷을 찍어 `안티그래비티\screenshot\` 디렉토리에 고해상도로 물리 저장합니다.
6. **세션 유지 원칙:**
   * 도메인 삭제 완료가 입증되면, 차장님께서 내장 뷰어 브라우저로 최종 결과를 눈으로 보실 수 있도록 브라우저 창을 닫지 않고 `browser.disconnect()`만 수행하여 세션을 종료합니다.
