## 1. 🚨 실서버 도메인 삭제 위험 전수 실사 및 정화 대책 (R2 개정 사유)

이전 R1 기획안 및 E2E 구동 과정에서 단순 `.first()` 선택자를 상속함으로써, 도메인 목록 첫 행에 위치한 QMS 메인 실서버 도메인(`shinwoo-valve-qms.vercel.app`)의 `Edit` 버튼을 마우스 커서가 엉뚱하게 클릭 타격하는 치명적인 인프라 파괴 결함이 검출되었습니다. 

다행히 전민재 차장님의 기민하고 신속한 브라우저 강제 셧다운(Shutdown) 대응 덕분에 실제 물리 삭제 대참사는 완벽하게 방지되었습니다. 이에 대한 깊은 반성과 함께 메인 실서버 도메인을 절대로 침범하지 않는 **초정밀 타깃 카드 필터링 안전장치**를 긴급 설계하여 기획안을 R2로 전격 개정합니다.

### 🛡️ 메인 서버 오작동 삭제 0% 원천 차단 필터링 알고리즘
1. **카드 전수 스캔:** `Edit` 버튼을 지니고 있는 Vercel 도메인 목록의 모든 행/카드 `div` 엘리먼트들을 전수 로딩합니다.
2. **타깃 도메인명 팩트 체크:** 각 행 카드의 내부 innerText를 검출하여, 오직 삭제 타깃인 **`shinwoo-qms-test.vercel.app`** 텍스트를 정확하게 포함하고 있는 **단 하나의 타깃 카드 행**만을 루프 조사를 거쳐 100% 색출합니다.
3. **자식 노드 고립 제어:** 색출된 타깃 카드의 컨테이너 내부로 로케이터 범위를 한정(`targetCard.locator(...)`)하여, 그 안에 소속된 `Edit` 버튼만을 고립 추적하고 마우스 궤적 이동 및 클릭을 타격합니다. 
4. 이 필터링 알고리즘 적용 시 메인 실서버 도메인(`shinwoo-valve-qms.vercel.app`) 카드는 물리적으로 선택 범위에서 **영구 완전 배제(0% 위험성)**됩니다.

## 3. 🛠️ 제안하는 정석 CDP 9222 삭제 E2E 프로세스

### 가. 대상 도메인 및 Vercel 관리 화면 URL
* **삭제 대상 도메인:** `shinwoo-qms-test.vercel.app`
* **Vercel 설정 탭 주소:** `https://vercel.com/mjjeon-min-jeas-projects/shinwoo-valve-qms/settings/domains`

### 나. 마우스 궤적 노출 및 삭제 구동 단계
1. **CDP 9222 연동 및 탭 포커스:** `connectOverCDP`로 직결하고 Vercel 도메인 설정 탭을 활성화합니다.
2. **파란색 마우스 원 커서 DOM 인젝션:** 브라우저 DOM 내부에 반투명 파란색 원(`rgba(0, 120, 255, 0.7)`) 커서를 동적 주입합니다.
3. **정밀 타깃 카드 검출:** 
   * `Edit` 버튼을 가진 모든 카드를 수집한 후, 내부 텍스트에 `shinwoo-qms-test.vercel.app`이 적혀 있는 타깃 카드를 전수 루프로 색출하여 `targetCard`로 확정합니다.
4. **부드러운 등속 마우스 LERP 이동 및 삭제 클릭:**
   * 검출된 `targetCard` 내부의 `Edit` 버튼으로 마우스 원 커서를 스르르륵 부드럽게 LERP 이동시켜 클릭 피드백(클릭 시 빨간색 반짝임)을 주고 클릭합니다.
   * 드롭다운 메뉴의 `Remove` 버튼으로 커서를 부드럽게 이동하여 클릭합니다.
   * 최종 삭제 확인 모달 팝업이 뜨면 빨간색 `Remove` 확인 버튼을 다시 한번 마우스 클릭 타격합니다.
5. **세션 영구 보존 마감:** `browser.close()` 없이 소켓만 끊어, 차장님 눈앞에 도메인이 정상 삭제 완료된 설정 화면 상태 그대로 온전히 영구 보존시킵니다.
