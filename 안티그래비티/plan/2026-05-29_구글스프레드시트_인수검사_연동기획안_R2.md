## 🔄 피드백 루프 안전장치 (Feedback Loop)

Gemini API 및 원격 시트 통신의 오류 가능성에 대비해 철저한 **4단계 피드백 루프**를 설계합니다:
1. **감지 (Detection):** 구글 시트 로드 실패, Gemini API 스로틀링(Throttling), Supabase 트랜잭션 에러 발생 시 즉각 예외를 Catch합니다.
2. **기록 (Logging):** 에러의 원인, 발생 시점, 대상을 DB 내 `system_logs` 테이블 또는 로컬 콘솔 로그에 정밀하게 기록합니다.
3. **경보 (Alert):** 동기화 오류 발생 시 QMS 메인 대시보드 및 인수검사 화면 상단에 **[⚠️ 구글 동기화 일시 오류 발생 (이전 저장된 캐시 데이터 표시 중)]** 경보 배너를 미려하게 노출하여 차장님께 즉각 공유합니다.
4. **결재 (Approval/Manual Sync):** 오류 상태에서 차장님께서 대시보드의 **[수동 즉시 동기화 및 재시도]** 버튼을 통해 재연동 명령을 인가할 수 있는 결재형 컨트롤 인터페이스를 제공합니다.

```
[1단계: 로컬 포트 가상 시뮬레이션] ➔ [2단계: 테스트웹(Staging DB) 배포] ➔ [3단계: 메인웹 최종 적용]
```

### 1단계: 로컬 가상 검증
* **검증:** `npm run dev` 로컬 Vite 서버 상에서 수동 API 호출을 모의 구현(Mocking)하여 UI 그리드 및 조치율 분석 차트의 색감/정렬 1차 점검.
* **롤백 방안:** 로컬 코드 변경 사항이 비정상일 경우, Git 로컬 체크아웃(`git checkout src/`)을 수행하여 1초 만에 즉시 원복합니다.

### 2단계: 테스트웹 배포 검증
* **검증:** GitHub 테스트 저장소(`qms.git`) ➔ 테스트 Vercel(`shinwoo-valve-qms.vercel.app`) ➔ Supabase 스테이징 DB(`srzaanvojyhwzugoaimk`) 연동 후 실제 구글 시트 원격 동기화 작동 여부 2차 크로스 검수. (Hobby 웹 + GitHub Actions Cron 모의 운용)
* **롤백 방안:** 스테이징 서버 동작 비정상 시, Vercel 어드민 콘솔에서 **"이전 성공 빌드 버전(Previous Deployment)"으로 즉시 원클릭 롤백** 처리합니다.

### 3단계: 메인웹 최종 배포
* **환경:** 6월 Vercel Pro 업그레이드와 동시 가동. GitHub 실서버 저장소(`qms-v2.git`) ➔ 메인 Vercel(`shinwoo-valve-qms-v2.vercel.app`) ➔ 실서버 Supabase DB 배포 및 전체 운영계 사이트 최종 개방.
* **롤백 방안:** 실서버 장애 감지 시, 메인 Vercel 즉각적 이전 빌드 롤백 실행 및 Supabase 마이그레이션 백업 파일(`db_backup.sql`)을 기동하여 복구 작업을 원자적으로 수행합니다. (파괴적 행위 발생 시 차장님 승인 필수 적용)

## 🧠 Gemini 2.5 Flash 코어 프롬프트 설계서 (Prompt Design)

스프레드시트의 비정형 `부적합 유형` 명칭들을 QMS 표준 카테고리로 초정밀 매핑하기 위한 프롬프트 구조입니다.

### 1. System Instruction (시스템 지침)
```text
너는 신우밸브주식회사 품질보증부의 인수검사 데이터 정제 전문가이다.
제공되는 입고/검사 데이터의 [부적합 유형] 텍스트를 정밀 분석하여, 아래 정의된 JSON 스키마 규격에 맞춰 정확히 매핑된 데이터만을 반환해야 한다. 절대 설명이나 마크다운 태그를 붙이지 말고 순수 JSON만 반환하라.

[부적합 유형(defectCategory) 매핑 규칙]
- 외관 불량, 흠집, 도장 불량, 사출 들뜸, 외관부적합 ➔ "외관부적합"
- 나사산 가공 불량, 리머 가공 오류, 홀 누락, 조립부 가공 오차, 가공부적합 ➔ "가공부적합"
- 치수 미달, 공차 초과, 금형 변형, 주물치수부적합 ➔ "주물치수부적합"
- 조립 뻑뻑함, 부품 누락, 유격 오류, 조립부적합 ➔ "조립부적합"
- 재질 상이, 성적서 불일치, 인장 강도 미달, 재질부적합 ➔ "재질부적합"
- 해당 없음, 합격, 빈 문자열 또는 공란 ➔ "합격"
- 그 외 어떤 매핑 규칙에도 속하지 않는 경우 ➔ "기타"
```

### 2. 입력 예시 (Few-shot)
* **Few-shot 1**
  * **입력:** `부적합 유형: 주물 치수 부적합`
  * **출력 JSON:**
    ```json
    {
      "defectCategory": "주물치수부적합"
    }
    ```
* **Few-shot 2**
  * **입력:** `부적합 유형: 가공부적합`
  * **출력 JSON:**
    ```json
    {
      "defectCategory": "가공부적합"
    }
    ```
* **Few-shot 3**
  * **입력:** `부적합 유형: 외관부적합`
  * **출력 JSON:**
    ```json
    {
      "defectCategory": "외관부적합"
    }
    ```
* **Few-shot 4**
  * **입력:** `부적합 유형: 사출 들뜸`
  * **출력 JSON:**
    ```json
    {
      "defectCategory": "외관부적합"
    }
    ```
* **Few-shot 5**
  * **입력:** `부적합 유형: (공백 또는 입고확인완료 등 내용없음)`
  * **출력 JSON:**
    ```json
    {
      "defectCategory": "합격"
    }
    ```

### 📋 출력 JSON 스키마 규격 강제 (Structured Output Schema)
```json
{
  "type": "object",
  "properties": {
    "defectCategory": {
      "type": "string",
      "enum": ["외관부적합", "가공부적합", "주물치수부적합", "조립부적합", "재질부적합", "합격", "기타"]
    }
  },
  "required": ["defectCategory"]
}
```
