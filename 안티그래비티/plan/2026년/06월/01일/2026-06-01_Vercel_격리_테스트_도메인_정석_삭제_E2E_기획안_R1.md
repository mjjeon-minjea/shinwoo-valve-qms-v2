## 1. 🚨 차장님 마스터 피드백 및 결함 정화 (핵심 원인과 대책)

이전 R0 기획안 수행 및 2차 시연 실패 과정에서 발생한 세션 폭파(프리징) 대참사와 30초 대기 붕괴 결함에 대해, 전민재 차장님께서 직접 정답과 마스터 피드백을 내려주셨습니다. 이를 기획안에 100% 반영하여 수립하였습니다.

### 가. 기술적 원인
* 안티그래비티의 내장 브라우저(CDP 9222 포트)를 공유하여 원격 제어하는 환경에서, 기존 플레이라이트 코딩 관행대로 스크립트 말단에 집어넣은 `browser.close()`가 실행되면서 호스트 세션 전체를 강제로 닫고 종료시켰던 것이 핵심 원인입니다. 이로 인해 브라우저 창이 꺼져 뷰어가 먹통이 되었습니다.
* 또한, `page.screenshot` 실행 시 폰트 및 외부 무거운 리소스 로딩 대기와 중첩되어 플레이라이트 내부적으로 30초 타임아웃을 유발하며 전체 스크립트가 멈추고 뻗어버리는 현상이 결합되었습니다.

### 나. 차장님 교시 해결 대책 및 안전망 장착
1. **종료 코드 전면 철거 (세션 보존):** 스크립트 마지막 줄의 `await browser.close();` 또는 `await browser.disconnect();` 구문을 완전히 제거합니다. 코드를 삭제하더라도 Node.js 프로세스가 종료되면 CDP 소켓 연결만 자연스럽게 해제되므로, 안티그래비티 대시보드 화면과 세션 탭은 그대로 멀쩡히 살아남아 보존됩니다.
2. **스크린샷 3초 타임아웃 통제:** 웹 폰트 로딩으로 인해 30초간 프리징이 걸리는 현상을 방지하기 위해, 모든 `page.screenshot` 호출부에 `timeout: 3000` (3초 타임아웃)을 명시하고 `try-catch` 블록으로 예외 처리를 감쌉니다. 스크린샷 과정에서 지연이 발생하더라도 3초 제한에 걸려 안전하게 스킵되므로, E2E 마우스 구동 시연과 레코딩 루프가 중간에 끊기지 않고 끝까지 무조건 완주됩니다.

```javascript
// [변경 전 예시] - 문제를 유발하는 구조
await page.screenshot({ path: 'screenshot.png' });
await browser.close(); 

// [검증 완료] 변경 후 예시 - 안전망 적용 및 세션 유지
try {
    await page.screenshot({ path: 'screenshot.png', timeout: 3000 });
} catch (e) {
    console.error("스크린샷 타임아웃 발생 (무시하고 진행):", e.message);
}
// browser.close() 구문은 의도적으로 완전히 삭제
```

## 3. 🛠️ 제안하는 정석 CDP 9222 삭제 E2E 프로세스

### 가. 대상 도메인 및 Vercel 관리 화면 URL
* **삭제 대상 도메인:** `shinwoo-qms-test.vercel.app`
* **Vercel 설정 탭 주소:** `https://vercel.com/mjjeon-min-jeas-projects/shinwoo-valve-qms/settings/domains`

### 나. 마우스 궤적 노출 및 삭제 구동 단계
1. **CDP 9222 연동 및 탭 포커스:** `connectOverCDP`로 직결하고 Vercel 도메인 설정 탭을 스캔하여 활성화합니다.
2. **파란색 마우스 원 커서 DOM 인젝션:** 브라우저 DOM 내부에 반투명 파란색 원(`rgba(0, 120, 255, 0.7)`) 커서를 동적 주입합니다.
3. **부드러운 등속 마우스 LERP 이동 및 삭제 클릭:**
   * `shinwoo-qms-test.vercel.app` 도메인 우측의 `Edit` 버튼으로 마우스 원 커서를 스르르륵 부드럽게 등속 이동(15단계)시켜 클릭 피드백(클릭 시 빨간색 반짝임)을 주고 클릭합니다.
   * 드롭다운 메뉴의 `Remove` 버튼으로 커서를 부드럽게 이동하여 클릭합니다.
   * 최종 삭제 확인 모달 팝업이 뜨면 빨간색 `Remove` 확인 버튼을 다시 한번 마우스 클릭 타격합니다.
4. **세션 영구 보존 마감:** 삭제 완료 즉시 `browser.close()` 없이 소켓만 끊어, 차장님 눈앞에 도메인이 정상 삭제 완료된 설정 화면 상태 그대로 완벽히 유지 보존시킵니다.
