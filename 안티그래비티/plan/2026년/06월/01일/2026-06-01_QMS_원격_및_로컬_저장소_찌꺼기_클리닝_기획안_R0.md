## 1. 개요 및 범위
QMS v2 프로젝트 환경의 정밀성을 확보하기 위해, 과거 개발 세션 및 테스트 라인에 누적되었던 미사용 찌꺼기 요소를 안전하고 확실하게 제거합니다.

- **범위:**
  1. **원격 테스트 저장소 (`test-origin`):** `test`, `staging` 등의 임시 브랜치 소거.
  2. **로컬 미추적 파일:** 수십 개 방치된 R0~R3 기획/결과 보고서 파일 안전 격리 및 컨펌 후 정리.
  3. **로컬 브랜치:** `temp-save`, `feature/3tier-isolation` 소멸.

---

## 2. 세부 작업 설계 (Action Plan)

### ① 원격 테스트 브랜치 정리 (test-origin)
- **대상 원격 브랜치:** `test`, `staging`, `feature/3tier-isolation`, `gh-pages`
- **적용 명령어:**
  ```powershell
  git push test-origin --delete test
  git push test-origin --delete staging
  git push test-origin --delete feature/3tier-isolation
  git push test-origin --delete gh-pages
  ```

### ② 로컬 미사용 브랜치 정리
- **대상 로컬 브랜치:** `temp-save`, `feature/3tier-isolation`
- **적용 명령어:**
  ```powershell
  git branch -D temp-save
  git branch -D feature/3tier-isolation
  ```

### ③ 로컬 Untracked 보고서 파일 찌꺼기 안전 정돈
- **안전장치:** 다이렉트로 삭제를 개시하지 않고, 우선 `안티그래비티/backup/` 폴더를 생성하여 모든 정리 대상 파일을 임시 이관(Move-Item)합니다.
- **최종 시정:** 차장님의 최종 리스트 검토 및 완전 삭제 승인을 구두/대화창으로 완전히 확보한 연후에 `Remove-Item`을 대행 가동시킵니다.

## 3. 검증 계획 (Verification Plan)

### ① 상태 정밀 진단
- `git branch -a` 명령을 가동하여 로컬 및 원격에 오직 보호 대상인 `main` 브랜치 계열만 무결하게 남았는지 인스펙션을 수행합니다.
- `git status -s` 명령을 구동하여 미추적(Untracked) 파일들이 온전히 백업 및 소거되었는지 검토 보고서를 작성합니다.
