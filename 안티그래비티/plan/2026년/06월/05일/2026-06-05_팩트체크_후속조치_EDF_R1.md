# 팩트체크 후속 조치 — [E] 테이블 재작업 + [D][F] 절차 보완

> **For agentic workers:** GEMINI.md §2 DNAS 결재 프로세스를 준수하여 차장님 승인 후 실행한다.

**Goal:** 팩트체크 보고서에서 적발된 [E] clone-db 테이블 불일치를 실측 스키마 기준으로 재작업하고, [D] DDL 미승인 절차를 명문화하며, [F] 비번 로테이션을 즉시 과업으로 처리한다.

- [ ] **Step 4: 복제 결과 대조 검증 (테이블별 메인↔테스트 행수 정합)**

```powershell
node -e "const d=require('dotenv');d.config({path:'.env.local'});const tables=['users','dev_notes','notices','weekly_reports','process_inspections','inspections','item_master','inquiries','suggestions','settings'];async function check(){for(const t of tables){const [m,s]=await Promise.all([fetch(process.env.PROD_SUPABASE_URL+'/rest/v1/'+t+'?select=*',{headers:{apikey:process.env.PROD_SERVICE_ROLE_KEY,Authorization:'Bearer '+process.env.PROD_SERVICE_ROLE_KEY,Prefer:'count=exact',Range:'0-0'}}).then(r=>r.headers.get('content-range')),fetch(process.env.VITE_SUPABASE_URL+'/rest/v1/'+t+'?select=*',{headers:{apikey:process.env.SUPABASE_SERVICE_ROLE_KEY,Authorization:'Bearer '+process.env.SUPABASE_SERVICE_ROLE_KEY,Prefer:'count=exact',Range:'0-0'}}).then(r=>r.headers.get('content-range'))]);console.log(t+' | main:'+m+' | test:'+s)}}check()"
```

Expected: 각 테이블의 메인/테스트 행수가 **동일**함을 확인.

> **메인 DB 쓰기 0건 증빙**: clone-db.js 내부의 Proxy 래퍼가 `insert`/`update`/`upsert`/`delete` 호출을 **물리적으로 throw**하므로, 메인 DB에 대한 쓰기는 코드 레벨에서 원천 차단됩니다.

- [ ] **Step 2: `.env.local`의 `PROD_SERVICE_ROLE_KEY`를 신규 키로 교체**

> service_role 키 확인: `Settings → API → service_role (secret)`

- [ ] **Step 3: 구키 무효화 확인**

```powershell
node -e "const d=require('dotenv');d.config({path:'.env.local'});fetch(process.env.PROD_SUPABASE_URL+'/rest/v1/users?select=count',{headers:{apikey:process.env.PROD_SERVICE_ROLE_KEY,Authorization:'Bearer '+process.env.PROD_SERVICE_ROLE_KEY,Prefer:'count=exact',Range:'0-0'}}).then(r=>console.log('Status:',r.status,'Range:',r.headers.get('content-range'))).catch(e=>console.error(e.message))"
```
