```
[AS-IS: 규칙-스킬 혼재 및 종속 구조]
- 규칙(Rules) 내부에 구체적인 물리 보관 경로(\안티그래비티\plan\)와 작명 규칙까지 장황하게 기술됨.

    ┌──────────────────────────────────┐
    │          GEMINI.md (규칙)         │ ➔ (물리적 경로 하드코딩 기술)
    └──────────────────┬───────────────┘
                       │ (동작 의존)
    ┌──────────────────▼───────────────┐
    │     revision-archiver (스킬)      │
    └──────────────────────────────────┘

[TO-BE: 규칙-스킬 완전 이격 및 매핑 구조]
- 규칙(Rules)에는 오직 "어느 시점에 어느 스킬을 가동한다"는 조건과 해당 스킬의 물리 경로만 심플하게 박아둠.
- 저장 경로, 백업, 리비전 제어, 오탈자 방지 등 실제 기계적 제어 로직은 전부 각 스킬 내부 정의서(SKILL.md)로 이식.
- 단일 책임 원칙(SRP)에 따라 텍스트 문서 리비전(revision-archiver)과 이미지 보관(image-archiver)을 완전 이격 분격함.

    ┌──────────────────────────────────┐
    │          GEMINI.md (규칙)         │
    │  - "Report 작성 시 스킬을 쓴다"    │
    │  - (경로: .agent/skills/...)     │
    └──────────────────┬───────────────┘
                       │ (호출/위임)
        ┌──────────────┴──────────────┐
        │                             │
    ┌───▼──────────────────┐      ┌───▼──────────────────┐
    │ revision-archiver   │      │ image-archiver       │
    │ (4대 텍스트 리비전)  │      │ (이미지 즉시 복사)    │
    │ - Plan, Task,        │      │ - 아티팩트 ➔ 로컬   │
    │   Walkthrough, Report│      │   images 폴더        │
    └──────────────────────┘      └──────────────────────┘
```
