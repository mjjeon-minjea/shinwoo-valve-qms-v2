# 📅 2026-03-15 대화 내역 요약

## 1. 분석현황 대시보드 구조 및 UI 전면 개편
* **작업장별, 공정별, 설비별, 모델대분류별 분석현황 개선**
    * 기존의 가독성이 떨어지는 막대 차트(검사 vs 불량 분포도)를 전면 철거.
    * 대신 화면 최상단에 **"불량률(Fail Rate)"** 기준 Top 10 (공정별은 Top 4) 위젯을 5x2 (또는 기타 비율) 그리드로 배치.
    * 위젯 시각화 시 링 게이지(Ring Gauge) 형태의 사각 카드를 커스텀 도입하여 PPM 임계값(100PPM, 1000PPM)에 따라 색상이 동적으로 변하도록 설정 완료.
    * 100%가 넘어가는 불량률 수치에 대응해 링 게이지가 역전되지 않고 최대 100%로 꽉 차게 렌더링되도록 차트 데이터 구조 보정.
    * `[ 조립반(A Line)-정수위밸브 ]`처럼 하이픈이 포함된 긴 문자열은 알맞게 잘라 서브타이틀로 줄바꿈 표시되도록 UI 디테일 조정.

## 2. 공정검사 엑셀 데이터 업로드 (Supabase 연동) 장애 조치
* **문제 증상:** "공정검사 이력 관리" 메뉴에서 엑셀 파일 업로드 시 `업로드 중 오류가 발생했습니다` 메시지 발생.
* **원인 분석:** 
    * 로컬 JSON DB에서 클라우드 Supabase 플랫폼으로 마이그레이션하며 `process_inspections` 테이블을 생성할 때, `inspectionType` (검사구분), `itemCode` (품목번호), `itemName` (품목명칭) 3개 컬럼의 스키마 명세가 누락되어 있었음.
    * 엑셀에서 추출한 해당 필드가 DB 테이블에 존재하지 않아 Insert 오류 발생.
* **해결 조치:**
    * Supabase SQL Editor를 통해 `ALTER TABLE process_inspections ADD COLUMN...` 쿼리로 3개 컬럼을 직통 추가하여 스키마 동기화 완료.

## 3. 업로드 건수 알림(Alert) 버그 픽스
* **문제 증상:** 엑셀 업로드를 성공적으로 마친 후 화면에 `undefined건이 업로드되었습니다` 표기 버그 발견.
* **원인 및 해결:** 
    * `api.js`의 `POST` 로직에서 단건이 아닌 `/batch` 삽입 시 응답 포맷이 Supabase API 정책상 데이터 배열 그대로가 넘어오는 문제.
    * 배열의 길이를 측정하여 `{ count: data.length }` 객체로 프론트엔드 반환 규격을 맞춤 강제 보정 완료.
