# 🔍 QMS 코드 리뷰 및 품질 검수 규칙 (qms-code-reviewer)

본 스킬은 에이전트가 작성한 프론트/백엔드 소스 코드의 품질을 검수하고, 차장님의 피드백을 수용하는 정형화된 상호 교류 규정입니다.

## 1. 코드 리뷰 요청 및 검수 (Requesting Review)
- 코드 작업이 완료되면, 에이전트는 작성된 소스 코드의 잠재적 오류를 먼저 정적으로 검수(ESLint 등)해야 합니다.
- 수정 범위 외의 멀쩡한 코드를 리팩토링하는 행위를 금지하며, 원본 코드의 컨벤션(인덴팅, 콤마 등)을 100% 모방하고 있는지 점검합니다.

## 2. 코드 리뷰 피드백 대응 (Receiving Review)
- 차장님의 반려 피드백이나 개선 요구가 수신되면, 에이전트는 맹목적으로 동조하거나 구차하게 사과하지 않습니다.
- 제시된 피드백의 기술적 의도를 정확히 파악하여 0.1초 내로 즉각 수정한 후, 수정 내역과 팩트 근거(파일 경로, 라인 등)를 차장님께 즉각 재보고합니다.
- 만약 기술적인 정합성 충돌이 발견될 경우, 사실(Fact)을 기반으로 정중하되 단호하게 교정된 의견을 건의합니다.
