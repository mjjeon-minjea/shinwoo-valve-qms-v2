# 📢 보고 및 사내 커뮤니케이션 규칙 (internal-comms)

본 규칙은 전민재 차장님께 보고를 올릴 때 가독성을 극대화하고 명료한 형식으로 문서를 작성하도록 규제하는 강제 규정입니다.

## 1. 대화창 가독성 극대화 및 요약 보고 [조항 2]
- **초간결 핵심 요약 보고**: 대화창 내 에이전트의 모든 설명 및 보고는 구차한 사과나 불필요한 텍스트 낭비를 배제하고 오직 핵심 내용만 가독성 있게 요약하여 초간결하게 작성합니다.
- **경로 링크만 제공**: 파일 생성 및 저장 완료 후 대화창에는 구구절절 코드를 적지 않으며, 오직 해당 파일로 이동할 수 있는 클릭 가능한 절대경로 마크다운 링크(예: `file:///` 절대경로 포맷)와 필요한 아티팩트(Artifact)만 깔끔하게 제공합니다.

## 2. 아티팩트(Artifact) 발행 규격 [조항 3]
- **발행 조건**: 에이전트가 제공하는 한글 텍스트 답변이 **300자 이상**이 되거나, **표(Table), 로직 구조, 다이어그램**을 출력하는 경우, 채팅창의 정돈성을 위해 반드시 아티팩트 또는 별도의 물리 마크다운 파일로 작성하여 제공해야 합니다.
- **보고서 아티팩트화**: 기획서(Plan)나 특별 분석 보고서(Report) 등의 공식 문서를 제출할 때는 사용자의 시각적 검토 편의성을 극대화하기 위해 대화창에 반드시 아티팩트(Artifact) 컴포넌트로 감싸서 전문을 함께 전송해야 합니다.

## 3. 사내 표준 문서 서식 (완료 보고서)
과업 완료 시 상신하는 최종 완료 보고서(Walkthrough)는 아래 4대 표준 서식을 엄격히 준수하여 작성해야 합니다.
1. **반영 내역 요약**: 수정한 파일들의 경로와 수정 목적을 리스트업합니다.
2. **반영 결과 요약**: 수정이 가해져 변경된 시스템 동작 결과를 간결히 기술합니다.
3. **검증 계획 요약**: 수정 사항을 검증하기 위한 자동 테스트 명령어 및 수동 검증 단계를 나열합니다.
4. **검증 결과 요약**: 실제 검증을 수행하여 획득한 로그, 테스트 패스 결과 등을 명시합니다.
