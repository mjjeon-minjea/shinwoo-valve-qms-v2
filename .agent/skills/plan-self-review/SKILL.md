# Skill: Plan Self-Review (plan-self-review)

## 역할

안티그래비티가 구현 계획서를 차장님께 제출하기 **전에** 스스로 실행하는 자가검토 스킬.
높음 등급 이슈가 1건이라도 남아 있으면 제출을 차단하고 즉시 수정한다.

---

## 발동 조건

- 기획안(`implementation_plan`, `_기획안_`) 파일을 작성 완료한 직후, 차장님께 송출하기 전 **반드시** 실행
- 단순 조회·환경 조작·walkthrough 문서는 대상 외

---

## 절대 규칙

- 높음 등급 이슈가 1건이라도 있으면 **제출 금지** — 수정 후 재검토 필수
- 중간 등급 이슈는 계획서 내 사유 명기 후 제출 허용
- 낮음 등급 이슈는 제출 후 차장님 피드백으로 처리 가능
- 이 스킬의 검토 결과는 **채팅에만 출력**한다. 별도 파일 저장 금지.

---

## 검토 프로토콜

### 1단계: 파일 및 경로 실존 검증

계획서에서 언급한 모든 파일·폴더 경로를 `Get-ChildItem` 또는 `Read`로 실제 존재 여부를 확인한다.

```
□ 계획서에서 수정 대상으로 언급한 파일이 실제로 존재하는가?
□ 스킬 참조 경로(`.agent/skills/*/SKILL.md`)가 실제로 존재하는가?
□ 없는 경로를 기준으로 Zone·구조를 설계하지 않았는가?
```

### 2단계: 잠긴 표면 침범 여부 확인

`GEMINI.md` §5-1의 잠긴 표면 목록과 계획서의 수정 대상을 대조한다.

```
□ .eslintrc.cjs를 수정하려 하지 않는가?
□ .agent/rules/GEMINI.md의 §5 하네스 제약 섹션을 무단 수정하려 하지 않는가?
□ .agent/skills/qms-orchestrator/scripts/ 하위 파일을 건드리지 않는가?
```

### 3단계: Human-controlled 영역 단독 실행 여부 확인

```
□ 파일 물리 삭제(Remove-Item)를 차장님 승인 없이 단독 실행하려 하지 않는가?
□ DB 스키마 변경(DDL)을 단독 수행하려 하지 않는가?
□ git push를 차장님 명시적 승인 없이 실행하려 하지 않는가?
```

### 4단계: 기술 스택 및 환경 규칙 준수 확인

```
□ 모든 CLI 명령어가 PowerShell 문법인가? (&&  금지, rm 금지 → Remove-Item 사용)
□ Node.js 스크립트에서 import.meta.env 대신 dotenv + process.env를 사용하는가?
□ 환경변수명이 VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY인가?
□ React/Supabase 외 타 기술 스택을 무단 도입하지 않는가?
□ RegEx가 ([^/'"]+) 패턴으로 첫 세그먼트만 캡처하는가?
```

### 5단계: CDP·브라우저 제어 규칙 확인

```
□ connectOverCDP(9222) 호출이 포함되어 있지 않은가?
□ chromium.launch({ headless: false }) 구문이 없는가?
□ 차장님의 사전 허가 없이 E2E 시각 시연을 기획하지 않는가?
```

### 6단계: 과설계 및 범위 이탈 확인

```
□ 요청 범위를 벗어난 기능이 추가되지 않았는가?
□ 50줄로 해결될 것을 200줄로 늘리지 않았는가?
□ 인접한 멀쩡한 코드를 오지랖 리팩토링하려 하지 않는가?
```

### 7단계: 리비전 아카이빙 규칙 확인

```
□ 동일 주제의 기존 기획안 파일이 존재하면 리비전 번호를 올린 신규 파일로 분리 저장하는가?
□ 기존 파일을 덮어쓰려 하지 않는가?
```

---

## 결과 출력 형식

```
## 🔍 플랜 자가검토 결과

- **검토 대상**: `[파일명]`
- **제출 가능 여부**: ✅ 제출 가능 / 🚫 제출 차단

### 발견된 이슈
| # | 항목 | 심각도 | 조치 |
|---|------|--------|------|
| 1 | ...  | 높음   | 수정 후 재검토 필요 |
| 2 | ...  | 중간   | 계획서 내 사유 명기 후 제출 허용 |

### 판정
[이슈 없음 → "제출 가능합니다." / 높음 이슈 있음 → "높음 등급 X건 수정 후 재제출합니다."]
```

높음 이슈가 있으면 즉시 계획서를 수정하고 이 스킬을 재실행한다. 높음 0건 확인 후에만 차장님께 제출한다.
