# 📝 QMS 계획 수립 및 실행 수트 규칙 (qms-planning-suite)

본 스킬은 QMS v2 프로젝트에서 기능 개발 및 버그 수정을 개시하기 전, 명확한 작업 로드맵을 정의하고 단계별 실행을 제어하기 위한 통합 스킬입니다.

## 1. 구현 계획서(Plan) 및 검토 보고서(Report) 규격 [조항 2]
- **Plan 작성 조건**: 구현 계획서(Plan)는 방향성이 완전히 합의되고 차장님께서 명시적으로 **"플랜 가져와라"**고 오더를 내리셨을 때에만 정식으로 `안티그래비티\plan\` 폴더에 수립합니다. (사전 독단 작성 금지)
- **문서 작성 전 규칙 선행 열람 의무**: Plan, Task, Walkthrough, Report 중 어느 것이든 작성하기 **직전에** 반드시 워크스페이스 규칙 `.agent/rules/GEMINI.md`의 **§4-1 도메인별 작성 스킬** 섹션을 확인하고, 해당 규칙에 지정된 도메인별 스킬을 호출한 뒤 그 절차를 따릅니다.
- **Report 작성 조건**: 대화창 출력 텍스트 총량이 **2000자**를 초과할 때, 또는 차장님이 명시적으로 **"분석해서 보고해줘"** 또는 **"검토해서 보고해줘"**라고 명령하실 때에만 특별 검토 보고서(Report)를 `안티그래비티\report\`에 작성하고 `revision-archiver` 스킬로 아카이빙합니다.

## 2. 파일 기반 계획 수립 지침 (Manus-style)
에이전트는 3단계 결재 문서를 수립하고 작업을 관리할 때 다음 원칙을 따릅니다:
- **계획 우선**: 복잡한 태스크를 시작하기 전 반드시 계획 파일을 구성합니다.
- **2-Action 규칙**: 2번의 보기/브라우저/검색 작업 후에는 반드시 핵심 발견 사항을 즉시 마크다운(findings)에 저장하여 컨텍스트 손실을 방지합니다.
- **의사결정 전 읽기**: 중요한 설계 결정이나 코드 수정을 하기 직전에 계획서를 다시 읽어 목표를 상기합니다.
- **행동 후 업데이트**: 각 작업 단계(Phase)를 완료한 후 즉시 계획서의 체크박스를 업데이트하고 파일 변경점 및 에러를 기록합니다.

## 3. 3단계 에러 극복 프로토콜 (3-Strike Error Protocol)
에러 발생 시 무분별한 동일 행동 반복을 금지하며 다음 단계를 밟습니다:
- **1차 시도**: 에러 로그를 면밀히 분석하고 원인을 찾아 정밀 픽스 적용.
- **2차 시도**: 동일 에러 지속 시, 다른 도구나 라이브러리를 사용하거나 아키텍처를 우회하는 대안적 방법 탐색 (절대 동일 실패 행동 반복 금지).
- **3차 시도**: 가설을 전면 재검토하고, 필요시 구현 계획서를 수정하여 다각도로 접근.
- **3회 실패 시**: 진행 상황 및 구체적 에러 내용을 정리하여 전민재 차장님께 보고하고 가이드를 청구.
