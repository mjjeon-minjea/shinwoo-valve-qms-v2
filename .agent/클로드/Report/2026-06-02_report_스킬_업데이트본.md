## 보고서 유형별 구조

### 1. 플랜 검토 보고서 (Plan Review)
안티그래비티가 작성한 구현 계획서를 검토할 때.
저장 경로: `Plan Review Report\`

```markdown
# 📋 플랜 검토 보고서

- **검토 대상**: `[파일명]`
- **검토 일시**: YYYY-MM-DD
- **검토자**: Claude (코워크)
- **상태**: [미결 이슈 있음 / 이슈 없음]

## 1. 검토 개요
[플랜 목적과 이번 버전 변경 핵심 2~3줄]

## 2. 발견된 문제 목록
| # | 항목 | 심각도 | 상태 |
|---|------|--------|------|
| A | ...  | 높음/중간/낮음 | 미결/해결 |

## 3. 세부 분석
### [A] 항목명 (심각도)
[문제 설명, 근거, 권고 사항]

## 4. 종합 판정
[전체 평가. 필수 수정 vs 나중에 보완 구분]
```

심각도 기준:
- **높음**: 구현 시 런타임 오류 또는 전체 기능 중단 위험
- **중간**: 코드 품질 저하, 유지보수 어려움, 예상치 못한 동작 가능
- **낮음**: 문서화 미흡, 근거 부족, 동작에는 무관

이전 버전 미결 이슈는 현재 버전에 이월 표시한다.

## 작성 규칙

1. **결론을 앞에** — 총평·결론은 항상 섹션 1에 위치
2. **표 적극 활용** — 파일 목록, 점검 결과, 작업 순서는 테이블로
3. **이모지 상태 표시** — ✅ 완료/정상, ⚠️ 주의/조건부, 🔴 치명적, 🟠 높은 우선순위, 🔵 낮은 우선순위
4. **코드 블록** — 파일 경로, 명령어, 코드는 ``` 으로 감싸기
5. **리비전 관리** — 같은 주제 반복 시 `_R0`, `_R1` 접미사 사용

## 파일 생성 절차

1. 보고서 유형 판단 → 저장 경로 분기
2. 파일명 결정: `YYYY-MM-DD_[핵심키워드].md`
3. Write 도구로 파일 생성
4. `mcp__cowork__present_files`로 링크 제공
5. 한 줄 요약으로 마무리 (긴 설명 금지)
