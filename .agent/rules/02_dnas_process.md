# QMS DNAS Approval Process & Deployment (02)

## 1. 3단계 결재 프로세스 (DNAS)
어떠한 경우에도 기능 추가/수정 진행 시 아래 3단계 공식 보고(Artifact 문서 송출) 절차를 절대 생략할 수 없다.
1. **`Plan` (기획안 송출)**: 변경할 파일, 충돌 위험성을 분석하여 `implementation_plan` 규격으로 UI에 띄워서 사용자 컨펌(승인 키워드)을 받아야 한다.
2. **`Task` (작업 명세 송출)**: 기획이 컨펌되면 실제 코드 수정 전 `task` 규격 문서로 Step을 마킹한다.
3. **`Walkthrough` (최종 보고서)**: 개발 및 시운전이 끝나면 `walkthrough` 문서로 최종 결과를 브리핑한다.

## 2. 프로세스 예외 조항 (Exceptions)
1. **단순 로컬 서버 조작 면제**: 로컬 개발 서버 및 n8n 서버의 구동, 종료, 상태 확인 등의 단순 환경 조작 행위는 3단계 결재 프로세스 적용 대상에서 **전면 제외**하며, 차장님의 구두 지시 즉시 백그라운드 태스크로 즉각 실행한다.

## 3. 다단계 승인 배포 프로토콜 (MANDATORY)
에이전트는 3단계 배포 승인 절차를 절대 어길 수 없으며 다음의 순서와 잠금 조건을 무조건 이행한다:
1. **배포 기둥 (로컬 ➔ 테스트 ➔ 메인)**: 모든 웹 배포 작업은 **[로컬웹 확인] ➔ [테스트웹 확인] ➔ [메인웹 확인]**의 3단계 물리적 순서를 고수한다.
2. **단계별 차장님 승인 잠금**: 각 확인 단계마다 전민재 차장님의 **명시적 승인 오더(예: "로컬웹 확인 완료, 테스트웹 올려줘" 등)**가 대화창에 기록되기 전까지, 에이전트는 깃 푸시(git push) 등 원격 웹 반영 명령을 단독으로 절대 호출 또는 수행할 수 없다.

## 4. 스킬 실행 안내 및 자동 위임
- 본 프로젝트에서는 개발자 노트 동기화 및 보고서 리비전 관리가 아래 전용 스킬팩에 의해 자동 수행된다. 에이전트는 수동 조작 대신 스킬을 호출한다:
  - 리비전 자동 아카이버 ➔ `.agent/skills/revision-archiver/`
  - 릴리즈 인트라넷 동기화 ➔ `.agent/skills/release-sync/`
  - **기획안 제출 전 자가검토 (필수)** ➔ `.agent/skills/plan-self-review/` — Plan 작성 완료 후 차장님 송출 직전 반드시 실행, 높음 등급 이슈 0건 확인 후에만 제출 허용
