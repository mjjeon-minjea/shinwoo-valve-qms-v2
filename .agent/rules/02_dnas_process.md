# QMS DNAS Approval Process

## 3단계 결재 프로세스 (MANDATORY)
어떠한 경우에도 기능 추가/수정 진행 시 아래 3단계 공식 보고(Artifact 문서 송출) 절차를 절대 생략할 수 없다.

1. **`Plan` (기획안 송출)**: 변경할 파일, 충돌 위험성을 분석하여 `implementation_plan` 규격으로 UI에 띄워서 사용자 컨펌(승인 키워드)을 받아야 한다.
2. **`Task` (작업 명세 송출)**: 기획이 컨펌되면 실제 코드 수정 전 `task` 규격 문서로 Step을 마킹한다.
3. **`Walkthrough` (최종 보고서)**: 개발 및 시운전이 끝나면 `walkthrough` 문서로 최종 결과를 브리핑한다.

> **⚠️ 경고:** 이 절차를 무시하고 채팅만으로 코어를 함부로 변경할 경우 룰셋 위반으로 간주된다.

## 📋 [P12] 개발자 노트 자동화 릴리즈 동기화 프로토콜
모든 메인 과업(Phase) 종료 시, AI는 아래의 절차를 자동으로 제안 및 실행해야 한다.

1. **[작성] 초안 게시**: 과업 완료 시 '원인-대책-결과' 및 '물리적 증빙'을 포함한 개발자 노트 초안을 작성하여 로컬 승인 대기열에 게시.
2. **[검토] 결재 대기 및 즉각 수정**: 차장님의 '승인/반려/삭제' 결재 대기. 반려 피드백이 들어올 경우, 지시사항을 즉시 반영하여 수정안 재게시.
3. **[확정] 공식 인트라넷 동기화**: 수정안이 최종 승인되면, 정식 문서 폴더(`docs/`)로 이동 확정 및 실서버 DB 게시판에 동기화(Insert/Update) 업로드.
4. **[검증] 최종 인스펙션 및 URL 보고**: 테스트웹 게시판에 정상 등록 및 노출되었는지 최종적으로 DB를 크로스체크 한 후 차장님께 릴리즈 보고.

## ⚠️ 프로세스 예외 조항 (Exceptions)
1. **단순 로컬 서버 조작 면제 (2026-05-26 차장 지시):**
   - 로컬 개발 서버 및 n8n 서버의 구동(npm start, node server.js 등), 종료(kill), 상태 확인 등의 단순 환경 조작 행위는 3단계 결재 프로세스(Plan, Task, Walkthrough) 적용 대상에서 **전면 제외**하며, 차장님의 구두 지시 즉시 백그라운드 태스크로 즉각 실행한다.
