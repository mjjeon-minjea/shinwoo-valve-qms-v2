# 📋 DNAS 결재 프로세스 (모든 에이전트 공통)

> 기능 추가/수정 시 아래 3단계를 절대 생략할 수 없습니다.

---

## 필수 3단계 결재 프로세스

### 1단계 — Plan (기획안)
- 변경할 파일, 충돌 위험성 분석
- `implementation_plan` 규격으로 UI에 표시
- **차장님 승인 키워드 수신 후** 다음 단계 진행

### 2단계 — Task (작업 명세)
- 실제 코드 수정 전 `task` 규격 문서로 Step 마킹
- 각 에이전트의 `status.md`에 진행 상황 기록

### 3단계 — Walkthrough (최종 보고서)
- 개발 및 시운전 완료 후 `walkthrough` 문서로 결과 브리핑
- 완료 후 해당 에이전트의 `memory.md`에 교훈 추가

---

## 에이전트 자율도 레벨

| 레벨 | 의미 | 적용 에이전트 |
|------|------|--------------|
| 1 | Read-only — 읽기·분석·보고만 | 관리자 에이전트 (보안상) |
| 2 | Draft — Plan→Task→Walkthrough 필수 ⭐ | 게시판, 인수검사, 공정검사, 업무관리 |
| 3 | 최종검사 에이전트 신규 설계 시 적용 | 최종검사 (설계 완료 후 결정) |

---

## 예외 조항
- 로컬 개발 서버 구동/종료 등 단순 환경 조작은 결재 프로세스 면제
- 단순 오타 수정, CSS 색상 변경은 Patch로 처리 (Plan 생략 가능)
