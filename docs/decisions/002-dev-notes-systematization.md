# QMS-DEC-002: 개발자 노트 시스템화 개선 의사결정서

## 1. 개요 (Context)
개발자 노트 관리 시스템의 DNAS(Data Network Approval System) Validator와 승인/반려 프로세스 간의 논리적 충돌로 인해 다음과 같은 세 가지 장애 요인이 발생하였습니다:
1. **DNAS 검증 꼼수 우회**: 반려 시 검증 오류를 우회하기 위해 본문에 품질 검증 키워드(원인, 대책, 결과, 물리적 증빙)를 자동 강제 삽입하여 기록 오염이 유발됨.
2. **배포일자 강제 덮어쓰기**: 차장님 최종 승인 시 무조건 현재 시간으로 배포일자가 강제 오버라이트되어, 과거 개발 완료 건에 대한 백데이트(과거일 지정) 정렬이 불가능해짐.
3. **한글 경로 빌드 크래시**: 로컬 PC 작업 경로 내 공백 및 한글 문자 인코딩 문제로 인해 Vite 빌드 시 무음 크래시 및 빌드 오류 발생.

---

## 2. 해결 방안 및 기술적 조치 (Action & Decision)

### A. DNAS 검증 예외 통제망 수립
- **원인**: 미들웨어 `server.js`에서 반려 요청에 대해서도 필수 검증 키워드(원인, 대책, 결과, 물리적 증빙)를 동일하게 강제하여 발생.
- **대책**: DNAS Validator 블록 첫 부분에서 `status === 'rejected'`일 경우 검증을 우회하고 즉시 `next()`를 호출해 넘기도록 예외 분기를 추가.
- **결과**: 반려 처리 시 품질 검증 통제망을 안전하게 우회하여 무오염 반려 사유만 온전히 기록할 수 있게 됨.

### B. 반려 처리 내 편법 우회 코드 전면 소거
- **원인**: 프론트엔드 `PostApproval.jsx`의 `handleReject` 내에서 DNAS 통과를 위해 본문에 꼼수 텍스트를 강제 결합해 저장하던 로직 잔존.
- **대책**: 해당 우회 텍스트 강제 삽입 로직을 완전히 제거하고, 순수한 반려 사유만 입력받아 패치하도록 정상화.
- **결과**: 데이터 위조 및 오염을 근본적으로 차단하고 반려 기록의 신뢰성을 회복함.

### C. 승인 시 날짜 수동 지정 미니 모달 구현
- **원인**: 승인 완료 시 배포 날짜가 오늘 시간으로 자동 고정되어 과거 버전 정렬이 불가능했던 구조적 한계.
- **대책**: `PostApproval.jsx`에 승인 클릭 시 배포일을 수동 선택할 수 있는 달력 입력창(미니 모달)을 적용. 입력값은 `compareVersions` 시맨틱 정렬을 보장하기 위해 `"YYYY-MM-DD 12:00"` 포맷으로 자동 정규화 처리.
- **결과**: 과거 배포 버전을 정합성 있게 정렬 및 소급 배포할 수 있는 유연성을 확보함.

### D. 담당자 표기 명칭 단일화
- **원인**: `item.manager`와 `item.author` 필드가 혼용되고 폴백 기본값(`AI`)이 제각각으로 표기되어 데이터 불일치 유발.
- **대책**: `item.manager` 관련 미사용 코드를 완전히 정리하고 `item.author`로 통합. 기본 폴백 담당자 이름을 `"Antigravity"`로 일치시킴.
- **결과**: 코드 및 데이터 모델의 복잡성을 낮추고 표기 일관성을 확립.
