# 신우밸브 QMS 프로젝트 일자별 상세 작업 총결산 일지 (Walkthrough)

이 문서는 프로젝트가 최초 생성된 2026년 1월부터 현재까지 매일 진행된 모든 세부 작업 내역을 낱낱이 기록한 **일자별 상세 마스터 일지**입니다.

---

## 🚀 Phase 1: 기반 시스템 구축 및 초기 세팅 (2026년 1월)

### 📅 2026-01-22 (프로젝트 최초 시작)
- **프론트엔드 뼈대 구축:** React(Vite) 기반의 신우밸브 QMS(품질관리 시스템) 프로젝트 최초 스캐폴딩 및 `Initial commit` 수행.
- **라우팅/디자인 초기 세팅:** 로그인 페이지, 사이드바 내비게이션, 그리고 Dashboard 메인 화면의 기초 레이아웃 생성.
- **버그 픽스:** `settings error` 및 Vercel 배포 시 발생하던 API 동적 라우팅(`VITE_SUPABASE_URL`) 환경 변수 에러 초도 대응.

### 📅 2026-01-26 (시스템 로직 점검 및 AI 룰 세팅)
- **AI 맞춤형 환경 설정:** 과장님의 지시에 따라 AI 에이전트(아스라다)가 무조건 100% 한국어로만 대답하도록 `Global Rule` 및 프로젝트 정책 수립.
- **로그인/권한 설정 문서화:** 향후 구현될 시스템의 역할(Role) 및 직급별(Rank) 권한 구조 기획.

## 🚀 Phase 2: 핵심 비즈니스 로직(검사 및 보고 기능) 개발 (2026년 2월)

### 📅 2026-02-02 (주간업무보고 구현)
- **주간업무보고 페이지 메인:** 부서원 목록이 표시되고, 이번주 주간업무보고 양식이 렌더링되도록 구현.
- **보고서 DB 통신:** Vercel에 연결된 DB와 프론트엔드가 데이터를 주고받을 수 있도록 JSON 서버 및 Supabase 사전 연결망 작업 진행.

### 📅 2026-02-03 (인수검사 자동화 및 브라우저 테스트)
- **인수검사(Inspection) 페이지 개설:** 납품된 자재의 품목, 검사 수량, 불량 수량, 결과를 추적하는 화면 개발.
- **자동화 브라우저 검증:** AI 서브 에이전트가 직접 브라우저를 띄워서 페이지 내 요소들을 클릭해 보고, 레이아웃에 이상이 없는지 기능 테스트 통과.

### 📅 2026-02-04 ~ 2026-02-06 (시스템 튜닝 및 데이터 병목 현상 해결)
- **대용량 삭제 버그 픽스:** 테스트 중 발생했던 서버 멈춤 현상(1만 건 데이터를 Delete할 때 브라우저가 다운됨)을 해결.
  - `Dashboard.jsx`의 `handleDeleteAll` 함수에 비동기 루프 기반 **1,000건 단위 Batch(분할) 삭제 모듈** 적용.
- **전역 System Check:** 코드베이스 전체를 진단하여 `eslint` 에러, 정의되지 않은 컴포넌트 호출 등 잠재적 오류를 일괄 리팩토링 및 클리닝 수행.

## 🚀 Phase 3: "공정검사" 신규 대규모 개발 및 대시보드 고도화 (2026년 3월)

### 📅 2026-03-09 ~ 2026-03-12 (공정검사 데이터 모듈 및 입력창 구현)
- **`ProcessHistory.jsx` 엑셀 데이터 파싱 고도화:**
  - MES 시스템에서 출력되는 현업 엑셀 데이터 중 "LOW DATA" 시트만 정확히 선별하여 파싱.
  - 새로운 "모델대분류 (Model Category)", "장비명" 항목도 매핑 키값에 추가하여 데이터베이스의 테이블 컬럼과 정확히 1:1 결합되게 함.
- **수동 입력 모달 개선:** 엑셀뿐만 아니라 사용자가 직접 검사 기록을 추가할 때 쓸 팝업 UI 내 항목(공정명, 조치사항 등)을 엑셀 스키마와 완벽 일치시킴.

### 📅 2026-03-14 (공정검사 대시보드 리디자인)
- **차트 및 목표치(Target) 최적화:** `ProcessInspectionDashboard.jsx` 내부에 수량 기준 및 건수 기준 부적합률(PPM 등)을 반원 링 형태의 도넛 게이지 차트로 전면 교체.
- **경고/시각화 로직 적용:**
  - 목표 수치(`100 PPM`, `0.01%`)를 초과하는 부적합 측정 시 차트 바늘/글씨가 자동으로 붉은색 경계 상태로 바뀌도록 구현.
  - 최상단 4대 핵심 KPI 카드 UI (총 검사 수량, 부적합률, 처리기입률, 미기입 건수) 신규 생성.
  - 우측 사이드 패널에 각 품질 지표별 심각/경고/안전 상태 리스트 렌더링.

### 📅 2026-03-15 (세부 분석현황 버그 패치 및 Supabase 100% 클라우드 이관)
- **조치율(Action Rate) 수식 보정:** 메인 대시보드 및 세부 분석 4페이지(`ProcessAnalysis`, `WorkplaceAnalysis`, `EquipmentAnalysis`, `ModelCategoryAnalysis`)에서 Excel 수식논리 그대로 "미기입 건수"와 "처리방안 기입률"이 정확한 비율 수치로 환산되도록 엑셀 계산식 이관 완료.
- **[Critical] Vercel 배포망 데이터 유실 에러 패치 및 전체 마이그레이션:**
  - 증상: 외부나 집에서 Vercel 호스팅에 접속 시 과거(1,2월) 입력해 뒀던 주간보고서 등 데이터가 보이지 않음.
  - 원인: `api.js` 코드가 Vercel 대신 로컬 PC의 `http://localhost:3001`을 바라보며 `LOCAL_JSON_TABLES` 규칙으로 차단하고 있었음.
  - 해결 1 (**클라우드 테이블 셋업**): 과장님을 통해 Supabase 대시보드에 빈 SQL 테이블 구조 새로 생성 (`process_inspections`, `weekly_reports`, `inspections` 등).
  - 해결 2 (**데이터 영구 이식**): 맞춤형 Node.js 스크립트(`migrate_to_supabase.js`)를 즉석 릴리즈 하여 로컬에 묶여 있던 과거 데이터 1만여 건을 Supabase로 손실 없이 100% 업로드 (Upsert).
  - 해결 3 (**연결선 절단**): `api.js`에서 로컬망 코드를 완전히 적출 후 깃허브 메인스트림(main)에 반영. Vercel 배포 자동 개시.
