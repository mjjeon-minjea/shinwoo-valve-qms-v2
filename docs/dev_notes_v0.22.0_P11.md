# [개발자 노트] v0.22.0 - P11: 초기 비밀번호 강제 변경 UI 구현

## 1. 개요 (Overview)
보안 취약점인 초기 비밀번호(`123456`) 사용자를 식별하여, 시스템 진입 전 비밀번호 변경을 강제하는 보안 인터셉터 UI 구현.

## 2. 기술적 트러블슈팅
### [이슈] 시스템 진입 보안 공백
- **원인**: 로그인 후 초기 비번 상태로 대시보드 접근이 가능하여 데이터 위변조 위험 노출.
- **대책**: `App.jsx` 최상단에서 사용자 상태를 감지하여 전용 모달(`PasswordChangeModal.jsx`)을 강제 라우팅으로 가로챔.
- **결과**: 초기 비밀번호 사용자의 대시보드 진입을 원천 차단하고 보안 정합성 확보.
