# Conversation Log - 2026-02-04 (Final)

## Summary

This session focused on debugging and refining the QMS application across three main areas: the Team Work Calendar, Profile Updates, and Weekly Report Permissions/UI.

## Key Accomplishments

### 1. Team Work Calendar (`CalendarView.jsx`)

- **Issue:** The calendar was displaying unapproved (draft/submitted) reports and sorting events incorrectly.
- **Fix:** Added a filter to only fetch `approved` reports. Adjusted sorting to prioritize "Vacation" (All Day) events, followed by time-based events.
- **Cleanup:** Identified and removed persistent "Ghost" test data (Vacation entry) from the production database via Vercel.

### 2. Profile Update Debugging (`api.js`, `App.jsx`, `Dashboard.jsx`)

- **Issue:** Profile updates failed with "Could not find the '0' column" error.
- **Root Cause:** `App.jsx` was sending `JSON.stringify`ed strings as the body, but `api.js` was treating them as objects (or vice versa in the backend), causing the payload to be interpreted as a string of characters rather than fields.
- **Fix:**
  - Updated `api.js` to automatically parse stringified JSON bodies if detected.
  - Refactored `App.jsx` and `Dashboard.jsx` to pass raw objects to `api.fetch`, ensuring consistency and clean code.

### 3. Weekly Report Permissions & UI (`WeeklyReport.jsx`, `App.jsx`)

- **Permissions:**
  - Corrected the hierarchy: **General (사원/대리)** -> **Manager (과장/차장)** -> **Director (부장/이사)**.
  - **Self-Review:** Allowed Managers (e.g., '전민재 과장') to review their own reports (transition to 'Reviewed').
  - **Approval:** Restricted 'Approve' button strictly to Directors ('손양수 부장'). Managers can no longer self-approve.
- **UI Status Labels:**
  - `submitted` -> **검토중**
  - `reviewed` -> **승인대기중**
  - `approved` -> **승인됨**
- **Demo Data:** Updated default users in `App.jsx` to reflect '전민재' as Manager (과장) and '손양수' as Director (부장) for proper demonstration.

### 4. Documentation

- Updated `presentation_outline.md` to reflect the corrected Role/Rank hierarchy.
- Verified that `WeeklyStatus.jsx` correctly filters for only 'approved' reports.

## Outcome

All identified bugs are resolved, and the application logic now aligns perfectly with the user's organizational structure and presentation requirements. All changes have been deployed to Vercel.
