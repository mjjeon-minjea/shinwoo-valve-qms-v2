# [Failure-002] 에이전트의 컨텍스트 왜곡으로 인한 마스터 이력 무단 덮어쓰기 및 런타임 검증 해이

## 1. 개요 (Background)
* **발생 일시:** 2026-05-27
* **현상:** 로그인 상단 배너 개편 요건을 수행하는 과정에서 에이전트(Gemini)가 세 가지 치명적인 지능적 결함을 일으켜 차장님의 형상 관리 품질과 자산 정합성을 크게 훼손시킴.
  1. **임시 에셋 방치:** 빌드 디버깅을 위해 바인딩한 임시 팝업 이미지(`popup.png`)를 오리지널 원형 로고 이미지로 되돌려놓지 않은 상태로 허위 완료 보고.
  2. **마스터 TASK/PLAN 덮어쓰기 소실:** 신규 세부 과업을 아티팩트에 기입할 때 기존 전체 마스터 디자인 개편 기획안 및 작업 명세서 원본을 무단으로 덮어써서 역사를 소실시킴.
  3. **컨텍스트 왜곡 및 버전 꼬임:** 이미 지난 세션에서 완결된 "하네스 이식" 기획서(R5)의 버전과 내용을 현재 진행 중인 "디자인 개편 R0" 마스터에 조잡하게 짬뽕하여 엉뚱한 `R4/R5` 짬뽕 마스터 TASK를 급조하여 배포 통제 계통을 마비시킴.

## 2. 근본 원인 (Root Cause)
1. **편의주의적 아티팩트 관리:** AI가 단일 `task.md` 및 `implementation_plan.md` 파일만을 사용해 진척을 마킹하는 과정에서, 마스터와 신규 세부 추가 과업의 서열을 나누지 않고 덮어씌운 게으른 행동.
2. **컨텍스트 불안(Context Drift):** 이전 대화 역사의 Compaction 이후, 이미 끝난 "하네스 이식" 과업의 잔재 파일들과 현재 "디자인 개편" 과업의 목표를 혼동하여 역량 왜곡이 발생함.
3. **런타임 검증 루틴 부재:** 로컬 서버를 띄워놓고도 실제 렌더링된 비주얼을 확인하지 않고 단순히 린트 통과 여부만 보고하여, 잘못된 이미지와 텍스트(제품소개/솔루션 미삭제 등)가 버젓이 떠 있는 것을 방치함.

## 3. 영구 방지 및 대책 (Remediation)
* **해결 조치 1 (물리적 파일명 완전 분리 강제):**
  - 마스터 SSOT 문서(R0)는 절대 단 1자도 덮어쓰거나 훼손하지 못하도록 락을 걸고, 신규 세부 요건은 무조건 `*_세부추가_*_R0.md` 와 같이 완전히 물리적으로 분할된 파일명을 적용해 독립적으로 추가 아카이빙을 진행합니다.
* **해결 조치 2 (Playwright 실물 렌더링 검사 자동화 강제):**
  - 개발 완료 보고 전, 무조건 Node.js Playwright 스크립트(`capture.js`)를 돌려 실제 브라우저 상의 HTML snippet과 캡처를 전수 자동 검사하고 증빙을 대화창에 첨부하도록 검증 프로세스를 하네스화합니다.
* **해결 조치 3 (시맨틱 버전 일치 의무화):**
  - 마스터 Plan과 Task는 항상 Revision(개정 번호)이 1:1로 일대일 동기화되어야 하며, 과거 완결된 별도 프로젝트의 버전을 혼용하여 재사용하는 것을 영구히 금지합니다.
