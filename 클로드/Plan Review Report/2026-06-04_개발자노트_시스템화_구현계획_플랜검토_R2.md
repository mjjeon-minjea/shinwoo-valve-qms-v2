## 3. 치명적 결함

이번 R1에는 **서버 기동을 막는 치명적 결함은 없다.** (직전 R1의 Task 1 결함 해소 확인.) 다만 실험 신뢰도를 무너뜨리는 1건을 아래에 격상해 기록한다.

🔴 **Task 5 — `Copy-Item -Exclude "node_modules" -Recurse`는 실제로 `node_modules`를 제외하지 못한다.**

- **문제 설명:** PowerShell `Copy-Item`의 `-Exclude` 파라미터는 `-Recurse`와 함께 쓰면 **하위 디렉터리를 신뢰성 있게 가지치기하지 못하는** 잘 알려진 한계가 있다. `-Exclude`는 열거되는 항목 이름에만 부분 적용되어, `node_modules` 디렉터리 자체로 재귀 진입해 내부를 그대로 복사하는 경우가 많다. 즉 이번에 고치려던 "node_modules 통째 복사" 문제가 그대로 재발할 수 있다.
- **재현/증상:** 복사가 수만 개 파일로 폭증하거나 Windows 260자 경로 제한에 걸려 중단 → "한글 경로 탓"이라는 **오진**으로 이어질 위험.
- **복구/대안:** `Copy-Item` 대신 `robocopy`로 디렉터리 단위 제외를 쓸 것.
  ```powershell
  robocopy "C:\Users\mjjeon\Desktop\QMS 프로젝트\shinwoo-valve-qms" "C:\Users\mjjeon\Desktop\qms_build_temp" /E /XD node_modules dist .git
  Copy-Item -Path "C:\Users\mjjeon\Desktop\QMS 프로젝트\shinwoo-valve-qms\.env.local" -Destination "C:\Users\mjjeon\Desktop\qms_build_temp\" -Force
  ```
  `robocopy`의 `/XD`(exclude directories)는 재귀 중에도 지정 디렉터리를 확실히 건너뛴다. 또한 `New-Item`은 폴더가 이미 있으면 에러를 내므로, 사전 정리(`Remove-Item ... -Recurse -Force` 또는 robocopy의 미러 옵션)를 병행할 것.

## 4. 권고 사항

🔵 **Task 3 — `handleExecuteApprove`에 `import.meta.env.DEV` 가드 복원.**
기존 `handleApprove`(원본 51~54행)에는 비-로컬 환경에서 차단하는 가드가 있었으나, 신규 `handleExecuteApprove`에는 빠졌다. 실서버 빌드에서 모달이 열리고 `최종 승인` 클릭 시 `localhost:3001` 접속이 네트워크 에러로 실패해 사용자에게 혼란을 준다. 함수 시작부에 아래를 복원할 것.
```javascript
if (!import.meta.env.DEV) {
    alert('승인 및 시스템 관리 권한은 로컬 인트라넷(localhost) 환경에서만 수행 가능합니다.');
    return;
}
```

🔵 **Task 3 — 저장 날짜 포맷 일관성.**
`finalDate = `${customDate} 12:00``는 `2026-04-13 12:00`처럼 공백 포함 비ISO 문자열을 만든다. 브라우저에서 대체로 파싱되긴 하나 표준이 아니며 기존 db.json 날짜(`2026-01-29`)와 형식이 다르다. ISO(`2026-04-13T12:00`) 또는 기존과 동일한 `YYYY-MM-DD`로 통일하는 편이 정렬·표시 일관성에 안전하다. (정렬은 주로 버전 우선이라 치명적이진 않음 — 낮음.)

🔵 **Task 3 — 구 `handleApprove` 데드코드 정리.**
버튼 이벤트를 모달 오픈으로 교체하면 기존 `handleApprove`(50~89행)는 호출되지 않는다. 혼란 방지를 위해 제거하거나 `handleExecuteApprove`로 일원화할 것.

## 6. 📌 안티그래비티에게 — 한 마디

> 실행 담당 안티그래비티에게.

이번엔 제대로 했다. Task 1, 실제 파일 열어보고 `server`로 고친 거 확인했고, 별도 미들웨어 안 만들고 기존 블록 안 content 검사 **앞에** 예외 끼워 넣은 것도 정확하다. 날짜도 `type="date"` 모달로 바꿔서 정렬 깨질 일 없애고, P4 이월도 명시했더라. 지적한 거 다 반영했다.

그런데 딱 하나, **Task 5의 `Copy-Item -Exclude "node_modules" -Recurse`는 함정이다.** 그거 PowerShell에서 재귀 복사할 때 node_modules를 제대로 안 빼준다 — 잘 알려진 버그다. 명령어는 그럴듯해 보여도 실제로 돌리면 node_modules가 통째로 따라온다. 그러면 또 경로 길이 터지고, 넌 "거봐 한글 경로 문제"라고 오진하게 된다. `robocopy ... /XD node_modules`로 바꿔라. 디렉터리 제외는 robocopy가 확실하다.

그리고 사소한 거 둘 — `handleExecuteApprove`에 `import.meta.env.DEV` 가드 빠뜨렸다(원본엔 있었음). 실서버에서 모달 누르면 localhost 붙으러 가서 에러난다. 복원해라. 또 옛날 `handleApprove`는 이제 안 쓰이니 지워라.

> **결론:** 통과다. Task 5 robocopy만 고치면 바로 실행해도 된다. 잘했다.
