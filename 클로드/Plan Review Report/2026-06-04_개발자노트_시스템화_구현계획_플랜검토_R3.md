## 4. 권고 사항

⚠️ **Task 3 — UI 안내 문구가 사실과 다르다 (코드는 안 고치고 문구만 바꿈).**
모달 안내문을 "`YYYY-MM-DD 12:00` 형식으로 **기존 날짜 데이터 정합성과 완전히 일치**하도록 저장"이라고 적었으나, 실제 `db.json`의 기존 58건은 전부 `2026-01-29`처럼 **시간 없는 날짜(`YYYY-MM-DD`)**다. 새 포맷 `2026-04-13 12:00`은 ` 12:00`이 붙어 기존과 **일치하지 않는다.** 코드(`finalDate`)는 R1과 동일하게 그대로다. 둘 중 하나로 정리할 것:
- (권장) 기존과 진짜 일치시키려면 `finalDate = customDate || (오늘 YYYY-MM-DD)` 로 시간 부분을 빼서 날짜만 저장, 또는
- 시간을 유지하려면 UI 문구에서 "완전히 일치" 표현을 빼고 "정렬 안전 포맷으로 저장" 정도로 정정.

기능상 정렬은 버전 우선이라 깨지지 않으므로 심각도는 낮음. 다만 "고쳤다고 적었는데 실제로는 문구만 바꾼" 형태이므로 정정이 필요하다.

🔵 **(정보성) Task 5 — `robocopy`의 종료 코드 해석 주의.**
`robocopy`는 정상 성공 시에도 **0이 아닌 종료 코드**를 반환한다(0~7=성공, 1=파일 복사됨, 8 이상=실패). 이번 과제 자체가 "Exit Code 1 빌드 크래시" 진단인 만큼, robocopy가 1을 뱉어도 그것은 **정상 복사**임을 혼동하지 말 것. 빌드 실패 판정은 `npm run build`(Step 3)의 종료 코드로만 판단해야 한다.
