## 3. 세부 분석

### [A] 히스토리 유지 + 회전 미실시 → 시크릿 영구 노출 (높음)

권장안(`git rm --cached`+`.gitignore`, 과거 히스토리 유지)은 **미래 커밋만** 차단한다. 회전을 안 하기로 한 이상, 이미 커밋된 `SUPABASE_SERVICE_ROLE_KEY`(RLS 우회 전체권한)·`PROD_DB_URL`/`STAGING_DB_URL`·`GEMINI_API_KEY`·Vercel 토큰은 `git show 55d2cfb~:.env.local.bak` 한 줄로 **누구나 복원 가능**한 상태로 영구 잔존한다.

- 플랜 문구 *"이후 커밋부터는 …더는 시크릿이 누출되지 않습니다"* 는 **오해 소지** — 정확히는 *"미래 누출만 차단, 과거 노출분은 그대로"*.
- **권고:** 이 잔존 위험을 User Review에 명시하고, 차장이 "회전·히스토리 정화 둘 다 안 함"을 **명시적으로 수용**하는 형태로 기록. service_role/DB/Gemini는 **장기 유효 키**라 Vercel OIDC(단기 토큰)와 위험도가 다름을 구분 적시.

### [B] `.gitignore` 등록 ↔ `rm --cached` 목록 불일치 (높음)

추적 중 파일은 `.gitignore`에 넣어도 추적이 안 풀린다(`rm --cached` 필요). 무시 목록엔 있는데 해제 목록엔 빠진 항목 3건:

| 항목 | `.gitignore` 등록 | `rm --cached` 목록 | 결과 |
|---|:---:|:---:|---|
| `src/data/mes_process_inspections.json` (406KB) | ✅ | ❌ 누락 | 추적 유지 → gitignore 무효 |
| `scratch/` 폴더(15개 추적) | ✅ `scratch/` | ⚠️ `query_…cjs` 1개만 | 나머지 14개 추적 유지 |
| `.obsidian/` 폴더(49개 추적) | ✅ | ✅ 폴더 전체 | 정합(단 [C] 참조) |

- **권고:** `mes_process_inspections.json`을 해제 목록에 추가(소스 import 없음 — 확인)하거나 무시에서 제거. `scratch/`는 "폴더째 해제"인지 "보안 1파일만 해제"인지 정해 범위 일치.

### [C] `.obsidian/` 일괄 무시가 위키 문서까지 해제 (높음)

`.obsidian/` 하위엔 에디터 설정뿐 아니라 **`.obsidian/wiki/history/` 13개 문서**(실제 지식베이스), `wiki/` 3개, 생성 캐시 `.obsidian/raw/graphify-out/`가 섞여 있다. 전체 무시+`rm --cached` 시 **위키 문서까지 추적 해제** → 플랜 Open Question #2("핵심 포맷만 남기고")와 **모순**.

- **권고:** 에디터 설정만 선별 무시(`.obsidian/.obsidian/`, workspace/appearance 등), `wiki/` 문서는 추적 유지 여부를 차장 확정. `.obsidian/raw/graphify-out/`는 캐시이므로 별도 무시 포함.

### [D] `vite.config.js` localhost 미포함 (중간)

`localhost:3001` 최대 밀집처는 `vite.config.js`(라인 14–23, proxy 10개)인데 중앙화 대상에서 **언급 없이 제외**됨. dev proxy라 유지할 여지는 있으나 범위 명시 필요.

- **권고:** vite proxy 의도적 제외 여부와 사유를 플랜에 1줄 명시.

### [E] 중앙화 시 DEV 가드 보존 (중간)

`DevNotes.jsx` GET은 `if (import.meta.env.DEV)`로 **dev만 localhost, prod는 `api.fetch`(Supabase)** 분기됨(확인). 종합보고서 §5-D의 "prod 미표출" 보류는 **해소**. 단 중앙화 시 config 기본값(localhost)이 prod 분기로 새지 않도록 **가드 안쪽에서만 적용** 필요.

- **권고:** 구현 시 `import.meta.env.DEV` 가드 반드시 보존.

### [F] `.env.example` 부재 (낮음)

`.gitignore`에 `!.env.example` 예외가 있으나 실제 추적된 `.env.example`는 없음(확인). 실 `.env*` 추적 해제 시 신규 개발자가 필요 키를 알 수 없음.

- **권고:** 값 없는 `.env.example`(키 이름만) 생성·추적 권장.
