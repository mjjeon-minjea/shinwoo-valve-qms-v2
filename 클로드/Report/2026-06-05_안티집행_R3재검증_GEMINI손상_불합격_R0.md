## 2. T2 불합격 상세 (핵심 이슈)
- **증상**: `.agent/rules/GEMINI.md`가 `file` 판정상 **`data`(바이너리)**. 텍스트가 아님.
- **원인 물질**: 파일 끝(164행)에 **NUL(0x00) 1,345바이트**가 패딩으로 주입됨. 총 15,850바이트 중 말미가 통째로 NUL.
- **줄바꿈**: CR 163개 = 거의 전 줄 CRLF. 직전에 적용한 `.gitattributes(eol=lf)` 정규화에서 벗어난 채 저장됨.
- **추정 경로**: T2 편집 시 PowerShell 출력 인코딩 사고(예: `>`/`Out-File` 기본 UTF-16LE 저장 잔재). 의도한 텍스트 위에 인코딩 깨짐이 얹힘.
- **잠긴 표면 영향**: §5 본문 텍스트(5-1 잠긴표면 4종 목록, 5-5 등)는 보존 확인되나, **§5 섹션이 위치한 파일 끝에 NUL 쓰레기가 추가**되어 "§5 100% 무변경" 규칙은 **위반**. 파일 자체가 깨져 에이전트 엔진이 규칙을 정상 파싱하지 못할 위험.

### 안티 보고와의 불일치
- 안티 walkthrough는 "§5 git diff 변경 없음으로 무변경 물리 입증"이라 기재. 그러나 실측은 NUL 주입 손상. git diff가 binary로 떠 끝 패딩을 표면화하지 못한 것으로 보임. **walkthrough의 T2 검증은 무효.**

### 정상 텍스트 반영 자체는 맞음(참고)
- §0-1 교체 문구("전역 조항 2의 2·3·5항을 따른다") 존재 ✓, 구 Override/`planning_mode` 문구 제거 ✓, §4-1 트리거 서브불릿(플랜 가져와라/2000자/아티팩트 필수) 제거 ✓, 매핑 4줄 보존 ✓.
