## 1. 총평

`.agent/` 폴더 **구조 자체는 정상**. 다만 `skills/` 40개 전수 검사 결과 **즉시 로드 실패 항목 9건**(SKILL.md 손상 8 + 누락 1)이 확인되어 **조건부 통과**로 판정한다.

손상 8건은 모두 **본문·frontmatter는 멀쩡하고 파일 꼬리에 NUL(0x00)만 붙은 상태**라, 내용 재작성 없이 **꼬리 NUL 제거만으로 복구 가능**하다. 그 외 이름 규칙 위반·원본 중복·에이전트 적용 범위 문제가 추가로 존재한다.

> ⚠️ 코워크 마운트 뷰는 staleness 가능 → 아래 수치는 차장님 실터미널에서 교차확인 권장.

## 3. 치명적 결함

### 🔴 결함 A — SKILL.md 8개 NUL 바이트 손상

전수 검사에서 NUL 바이트가 **전부 파일 꼬리에 한 덩어리로** 발견됨(중간 산발 아님). 본문과 frontmatter는 모두 정상이며, NUL 제거 시 8개 모두 UTF-8 정상 디코딩됨.

| 스킬 | 크기(B) | NUL 수 | NUL 위치 | frontmatter | NUL 제거 후 |
|------|--------:|-------:|----------|:-----------:|:-----------:|
| brainstorming | 10,798 | 7,391 | 끝부분 뭉텅이 | 정상 | ✅ |
| dispatching-parallel-agents | 6,623 | 2,771 | 끝부분 뭉텅이 | 정상 | ✅ |
| image-archiver | 1,251 | 274 | 끝부분 뭉텅이 | 정상 | ✅ |
| plan-self-review | 4,418 | 25 | 끝부분 뭉텅이 | 정상 | ✅ |
| skill-creator | 33,653 | 15,529 | 끝부분 뭉텅이 | 정상 | ✅ |
| ui-ux-pro-max | 45,182 | 26,738 | 끝부분 뭉텅이 | 정상 | ✅ |
| writing-skills | 23,279 | 6,278 | 끝부분 뭉텅이 | 정상 | ✅ |
| xlsx | 11,754 | 379 | 끝부분 뭉텅이 | 정상 | ✅ |

**추정 원인:** 꼬리 zero-padding은 **파일 쓰기 중단 / 동기화 충돌**의 전형적 흔적(예: 저장 도중 OneDrive·Obsidian 동기화 또는 PowerShell 훅이 겹침). 이 repo는 과거에도 CRLF 오염·GEMINI 손상 이력이 있어 동일 계열 파일 쓰기 사고로 보임.

**복구 방법 (안티그래비티 집행 — 내용 보존, 꼬리 NUL만 제거):**

```powershell
# 실행 전 git commit 또는 백업 필수
Get-ChildItem ".agent\skills\*\SKILL.md" | ForEach-Object {
  $b = [IO.File]::ReadAllBytes($_.FullName) | Where-Object { $_ -ne 0 }
  [IO.File]::WriteAllBytes($_.FullName, $b)
}
```

**검증:** 복구 후 `Get-Content` 또는 `file` 로 8개 파일이 정상 텍스트(UTF-8)로 인식되는지, frontmatter `name:`·`description:` 파싱되는지 재확인.

## 4. 권고 사항

🟠 **(C) snake_case → kebab-case 통일** — 대상 4개: `dnas_process`, `tech_stack`, `harness_constraints`, `knowledge_archiving`. 폴더명·frontmatter `name`·GEMINI.md 참조 경로를 함께 변경해야 링크가 깨지지 않음.

🟠 **(D) 폴더명 ↔ name 일치** — `react-best-practices` 의 name을 폴더명과 맞추거나(`react-best-practices`), 폴더를 name(`vercel-react-best-practices`)에 맞춤.

🔵 **(E) 원본 중복 정리** — `tech_stack`, `dnas_process` 가 `_shared/*.md` 와 `skills/*/SKILL.md` 양쪽에 존재. 단일 진실 한 곳으로 통일하고 나머지는 참조 링크 처리. (GEMINI.md §1은 `skills/` 쪽을 가리킴)

🔵 **(F) rules 적용 범위 인지** — `.agent/rules/GEMINI.md`(`trigger: always_on`)는 **안티그래비티 전용**. 코워크(Claude)는 `.agent/`를 읽지 않고 루트 `CLAUDE.md`를 따름. 두 에이전트 규칙을 동일하게 유지하려면 양쪽을 각각 관리해야 함.

## 5. 결론

폴더 구조는 유지해도 무방. 우선순위는 **① 결함 A NUL 8건 복구 → ② 결함 B SKILL.md 작성 → ③ 권고 C 이름 통일** 순. 결함 A·B 해소 시 현재 로드 실패 중인 9개 스킬이 즉시 정상화된다. 모든 파일 수정은 안티그래비티가 집행하며, 집행 전 백업과 집행 후 재검증을 권고한다.
