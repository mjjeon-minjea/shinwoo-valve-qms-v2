## 1. 총평

**R0의 「결함 A — SKILL.md 8개 NUL 바이트 손상」은 오류였다. 전면 철회한다.**

안티그래비티의 반박(Node.js 바이트 단위 검사 결과 NUL 0개)이 정확하다. 마운트와 독립된 경로인 **git object store**로 교차검증한 결과, 8개 파일 모두 **커밋 blob 기준 NUL이 0개**로 확인됐다. 내가 보고한 NUL은 실파일이 아니라 **코워크 마운트 뷰의 staleness(꼬리 절단 + zero-padding) 산물**이었다. 이는 CLAUDE.md가 명시적으로 경고한 현상이며, 점검 착수 시 이를 교차확인하지 않은 것은 코워크(나)의 절차 위반이다.

추가로, **NUL 제거 스크립트를 실행하지 않은 판단이 옳았다.** 만약 실행했다면 마운트의 절단된 내용을 실파일에 덮어써 실제 본문 일부(파일당 수십~수백 바이트)가 영구 유실됐을 것이다(§2 크기 차이 참조). 안티그래비티의 집행 보류가 데이터를 지켰다.

## 2. 교차검증 결과 — 마운트 뷰 vs git 실 blob

| 스킬 | git blob NUL | git 크기(B) | 마운트 NUL | 마운트 크기(B) |
|------|---:|---:|---:|---:|
| brainstorming | **0** | 10,634 | 7,391 | 10,798 |
| dispatching-parallel-agents | **0** | 6,441 | 2,771 | 6,623 |
| image-archiver | **0** | 1,232 | 274 | 1,251 |
| plan-self-review | **0** | 4,304 | 25 | 4,418 |
| skill-creator | **0** | 33,168 | 15,529 | 33,653 |
| ui-ux-pro-max | **0** | 44,524 | 26,738 | 45,182 |
| writing-skills | **0** | 22,624 | 6,278 | 23,279 |
| xlsx | **0** | 11,463 | 379 | 11,754 |

**해석:** 전 파일 git blob NUL = 0(정상). 마운트 뷰는 실내용의 꼬리 일부를 누락한 뒤 NUL로 채워 표시했다 → 파일 손상이 아니라 **마운트 읽기 결함**. 검증 명령:

```bash
git cat-file -p HEAD:".agent/skills/<스킬>/SKILL.md" | tr -cd '\000' | wc -c   # → 0
```

## 4. 재발 방지 (코워크 자체 규율)

1. 파일 무결성·바이트 단위 판정은 **코워크 마운트로 단정하지 않는다.** `git cat-file` blob 또는 안티그래비티 실터미널 결과로만 확정한다.
2. 마운트에서 관측된 꼬리 NUL·크기 이상은 **진단 신호가 아니라 마운트 아티팩트로 우선 간주**한다.
3. CLAUDE.md 「마운트 staleness 교차확인」 주의 조항을 **점검 착수 전 필수 체크리스트**로 강제한다.
