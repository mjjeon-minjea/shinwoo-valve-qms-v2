# 집행 보류 판정 — 워킹트리 CRLF 대량오염 진단 R0

- **작성일**: 2026-06-05 / **작성**: 코워크 (마운트 bash 실측)
- **판정**: **NO-GO. 안티 집행 즉시 보류.** 워킹트리가 정상 상태가 아님.
- **요약**: 안티 보고와 실제 git 상태가 크게 불일치. 실제로는 **1,107개 파일이 줄바꿈(CRLF) 변경으로 통째 오염**. 안티 세션 출력도 깨짐.

---

## 1. 두 가지 문제

### 문제 A — 안티 세션 출력 손상
- 안티 답변 중간부터 `_ 100 _ 105 __ _ 110 ...` 줄번호 토큰이 수백 회 반복되는 garbage 출력 발생.
- 정상적 판단·보고가 불가능한 상태 → **안티 세션 재시작 권장.**

### 문제 B — 안티 git status 보고가 실제와 불일치
| 구분 | 안티 보고 | 실측 |
|------|-----------|------|
| Modified | "writing-plans 한글화 2개 등" | **1,107개 파일** |
| 변경 규모 | (언급 없음) | **+207,537 / -207,634 줄** |
| 성격 | "한글화·기획서 등 정당한 작업" | **거의 전부 줄바꿈(CRLF) 변경, 내용 동일** |

---

## 2. 변경 정체 = CRLF 줄바꿈 대량 재기록 (실측 확정)
- HEAD 원본 = **LF(`\n`)**, 현재 워킹트리 = **CRLF(`\r\n`)**. 바이트 비교 확인(`--- 0a` vs `--- 0d 0a`).
- 상위 40개 tracked 파일 전수 비교 → **40/40이 "줄바꿈만 다름, 내용 동일"**.
- `insertions ≈ deletions`(207,537≈207,634) = whole-file 줄바꿈 재기록의 전형적 신호.
- 원인 환경: `core.autocrlf` **미설정** + `.gitattributes` **없음** → 어떤 에디터/도구가 전 파일을 CRLF로 재저장한 것으로 추정.
- **GEMINI.md도 동일** — §5(잠긴 표면) 포함 전체가 diff에 잡히나, 줄바꿈만 차이(텍스트는 HEAD와 100% 동일).

---

## 3. 이 상태로 집행하면 안 되는 이유
1. **T2 §5 무변경 검증 불가**: GEMINI.md 전체가 이미 diff에 잡혀 있어 `git diff`로 "§5 0줄 변경"을 증명할 수 없음. 검증 게이트가 무력화됨.
2. **T2/T3 롤백 안전장치 손상**: `git checkout -- GEMINI.md` / SKILL.md 가 줄바꿈 오염 상태로 되돌아가, 깨끗한 원복이 아님.
3. **노이즈 커밋 위험**: 이 상태에서 무언가 커밋되면 20만 줄짜리 줄바꿈 노이즈에 실제 변경이 파묻혀 추적·리뷰·롤백이 사실상 불가.
4. T1(스킬 이동)은 줄바꿈과 무관하게 동작하나, 검증·diff가 오염된 환경에서 진행하는 것은 부적절.

---

## 4. stash는 부적절
- 1,107개 + untracked 14개를 stash하면 작업 산출물이 통째로 치워지고, **CRLF 원인은 미해결로 남아 재발**한다.
- untracked(플랜·보고서·`CLAUDE.md`)는 `-u` 없이는 안 치워지고, 붙이면 또 위험. 깔끔한 해법이 아님.

---

## 5. 권고 조치 (실터미널, 안티 또는 차장님)
**먼저 줄바꿈 오염을 정리해 워킹트리를 클린으로 만든 뒤 집행 재개.** 두 경로:

- **경로 A (가장 단순·되돌리기) — 줄바꿈 변경 폐기**
  `git checkout -- .` 로 tracked 파일을 HEAD(LF)로 원복.
  → 줄바꿈 오염만 사라지고 untracked(플랜·보고서)는 보존됨. 단 tracked에 섞인 "진짜 내용 변경"이 있다면 함께 사라지므로, 사전에 `git diff --stat`에서 줄바꿈 외 실변경 유무 확인 필수.
- **경로 B (정석) — 줄바꿈 정규화 후 커밋**
  `.gitattributes`에 `* text=auto eol=lf` 추가 → `git add --renormalize .` → 커밋으로 LF 고정. 이후 재발 방지.

부가 확인 사항:
- `HANDOFF.md` **삭제됨(D)** — §5 규칙상 `shinwoo-system\` 보관 대상. 의도된 삭제인지 확인.
- `CLAUDE.md` **untracked 신규 생성** — §5-3 체크리스트 "CLAUDE.md 생성/참조 금지(GEMINI.md로 대체)" 항목과 충돌 가능. 정체 확인 필요(코워크 세션 생성물일 수 있음).
