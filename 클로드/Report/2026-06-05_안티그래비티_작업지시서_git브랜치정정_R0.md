# 🛠️ 안티그래비티 작업 지시서 — git 브랜치 정정 + 통합본 main 반영

> **수신:** 안티그래비티 (모델: Claude Opus 4.6)
> **발신:** 코워크(검토) · 승인: 전민재 차장님
> **작성일:** 2026-06-05
> **배경:** 로컬 작업 브랜치가 `main`(=메인웹/프로덕션 연동)인 채로 개발 → staging에만 커밋 → `git checkout main`에서 작업분 소실. 브랜치 운영 모델을 DNAS 3단계(로컬→테스트→메인)와 일치시킨다.
> **성격:** 실행 지시. 범위 안에서만 작업하고, 프로덕션 반영은 차장님 승인 게이트를 지킨다.

---

## 0. 전제·제약 (작업 전 필독)

1. **배포 3단계 잠금 (전역 조항7 · 워크스 §2-3):** `main` push = **프로덕션(메인웹) 배포**. 차장님의 **명시적 승인 오더("메인 올려줘" 등)** 전에는 `main` push를 **단독 수행 절대 금지**.
2. **재커밋 금지 — merge만:** staging의 변경을 main에 **따로 다시 커밋하지 않는다.** 히스토리 분기(divergence) 방지를 위해 반드시 `git merge`로 가져온다.
3. **PowerShell 전용**, `&&`·`rm` 금지. 크롬 좀비/CDP 금지(조항8).
4. **마운트 staleness 주의:** 모든 git 상태는 **실터미널에서 직접** 확인(코워크 마운트 신뢰 금지).
5. 의심·충돌 시 즉시 중단·보고.

---

## 1. 작업 범위

| Task | 내용 | 담당/게이트 |
|------|------|------------|
| T1 | 현황 확인 (브랜치·diff 조회) | 안티 (조회, 즉시) |
| T2 | 로컬 작업 브랜치를 `staging`으로 전환 | 안티 (로컬, push 없음) |
| T3 | 통합본을 `main`에 merge + push | **차장님 승인 게이트 (프로덕션)** |

---

### Task 1 — 현황 확인 (조회만, 변경 없음)
```powershell
git branch -a
git status
git log --oneline -5 main
git log --oneline -5 staging
git diff --stat main staging
```
**보고 항목:** ① 현재 체크아웃 브랜치 ② `main`↔`staging` 차이 파일 목록 ③ **GEMINI.md 외에 다른 변경(앱 코드 등)이 섞여 있는지** 여부.
> 목적: main에 올라갈 변경이 "의도한 것(통합 GEMINI.md 등)만"인지 사전 식별. 섞여 있으면 T3 보류·보고.

---

### Task 2 — 로컬을 staging으로 전환 (로컬·비파괴·push 없음)
```powershell
git stash list           # 미저장 작업 잔존 확인 (있으면 보고)
git checkout staging
```
- 이후 **모든 일상 개발은 `staging` 브랜치에서** 한다. 로컬 vite 개발서버는 체크아웃 브랜치 코드를 띄우므로 "로컬=staging" 일치.
- `main`은 건드리지 않는다(운영 배포 시에만 merge로 수령).

---

### Task 3 — 통합본 main 반영 (⚠️ 차장님 승인 게이트 = 프로덕션 배포)

**선결 3조건 모두 충족 시에만 착수:**
1. staging 웹에서 통합본 동작 **검증 완료**
2. T1 diff 검토로 **main에 올릴 변경이 의도한 것만**임을 차장님이 확인
3. 차장님 **명시적 승인 오더**("메인 올려줘" 등) 대화창 기록

**방법 (재커밋 금지 — merge):**
```powershell
git checkout main
git merge staging          # staging 이력 그대로 병합 (cherry-pick/재커밋 금지)
git push origin main       # 위 3조건 충족 + 승인 후에만
git checkout staging       # 작업 브랜치로 복귀
```
- merge 충돌 발생 시 **즉시 중단·보고** (임의 해결 금지).
- `main`에 직접 파일 편집·재커밋 금지.

## 3. 절대 금지 / 게이트 요약
- 차장님 명시 승인 **전** `main` push 금지 (프로덕션).
- staging→main **재커밋(중복 커밋) 금지** — merge만.
- diff에 **의도치 않은 변경**이 섞여 있으면 push 보류·보고.
- merge 충돌 임의 해결 금지.
