## 1. 총평

**조건부 진행 가능.** 핵심 원리는 명확하다 — `always_on` 규칙 파일에는 **모든 행동을 제약하는 가드레일**만 남기고, **특정 순간에만 작동하는 절차·서식·체크리스트**는 해당 스킬의 `SKILL.md`로 옮긴다(스킬은 필요 시에만 로드되므로 상시 컨텍스트 비용이 줄고 가독성이 오른다). 다만 두 가지 차단 요인이 있다: **(1) §5 전체가 잠긴 표면**이라 코워크·안티가 손댈 수 없고 오직 차장님만 편집 가능, **(2) §5-4 Rule 3("신규 규칙은 본 GEMINI.md에만 추가")이 추출 방향과 정면 충돌**하므로 차장님이 이 룰을 먼저 개정·허가해야 추출 자체가 하네스 위반이 아니게 된다. 가드레일(§1, §2-3, §5-1, §5-2)은 **추출 금지** — 스킬 로드가 실패해도 안전장치는 살아 있어야 한다.

## 3. 추출 대상 상세 (우선순위순)

### 🟠 1순위 — §3 지식 아카이빙 → `knowledge-extractor` SKILL
`knowledge-extractor/SKILL.md`는 현재 24줄에 불과한데, 실제 추출 기준(4대 카테고리·3대 강제원칙·파일명 과거날짜 규칙, §3-1·§3-2)은 GEMINI.md에 상시 적재돼 있다. 이 내용은 "대화 저장 지시" 순간에만 쓰이는 순수 절차다. 전량 SKILL.md로 이전하고 GEMINI에는 **"지식 자산화 지시 시 → `knowledge-extractor` 호출"** 한 줄만 남긴다. 잠긴 표면 아님 → 안티 실행 가능.

### 🟠 2순위 — §4 서식·연계 세부 → 각 작성 스킬
§4-1의 "한글 DNAS YYYY-MM-DD…" 서식 규격, 저장 폴더, 형식 지정은 각 스킬(writing-plans / planning-with-files / internal-comms / doc-coauthoring)의 SKILL.md로 내린다. §4-3 표준 연계 순서는 §2-1과 동일 내용이므로 **한 곳(스킬)**에만 둔다. GEMINI에는 트리거(언제 호출)만 표로 유지. 잠긴 표면 아님.

### 🟠 3순위 — §5-3 / §5-5 → `plan-self-review` SKILL  ⚠️ 잠김
기획안 제출 전 체크리스트(§5-3)와 자가검토 절차(§5-5)는 `plan-self-review` 스킬이 실행하는 바로 그 내용이다. SKILL.md(현재 109L)로 흡수하고 GEMINI에는 게이트 선언("High 0건 전 송출 차단") 1줄만. **단 §5는 잠긴 표면 → 차장님 직접 편집만 허용.**

### 🔵 4순위 — §1-3 SemVer / §5-4 절차성 룰
버전 규칙(§1-3), 사전점검 명령(§5-4 Rule1·2), 리비전 증가 규칙(Rule5)은 레퍼런스/절차라 추출 가능하나 분량이 작아 효과는 제한적. Rule5는 `revision-archiver`로 자연 귀속. (§5-4도 잠긴 표면)

## 4. 권고 사항

1. **추출 금지 가드레일 확정**: §1-1·1-2·1-4, §2-2·2-3, §5-1·5-2는 안전장치이므로 어떤 경우에도 always-on 유지. 스킬 로드 실패 시에도 살아 있어야 함.
2. **잠긴 표면 절차 분리**: §5-3·5-4·5-5 이동은 **차장님 본인만** 가능(§5-1). 안티에 지시 불가 — 코워크가 개정안 텍스트만 작성해 드리고 차장님이 직접 반영.
3. **선결 룰 개정**: §5-4 Rule 3("신규 규칙은 GEMINI.md에만 추가")을 "가드레일은 GEMINI.md, 절차는 SKILL.md"로 차장님이 먼저 개정해야 추출이 합법.
4. **전역 파일 대조 필수**: §0/Rule6이 전역↔로컬 카피 드리프트를 경고. 전역 `C:\Users\mjjeon\.gemini\GEMINI.md`를 받아 **중복 구간**을 먼저 식별해야, 전역이 담당할 내용을 워크스페이스에서 잘못 추출하는 실수를 막는다.
5. **단일 출처 원칙**: §2-1↔§4-1↔§4-3 3중 중복부터 일원화하면 추출 없이도 가독성이 즉시 개선됨.

## 5. 결론

워크스페이스 GEMINI.md 173줄 중 **약 §3 전체 + §4 세부 + §5-3/5-5**가 스킬로 이전 가능하며, 이는 always-on 부피를 눈에 띄게 줄인다. 그러나 절반은 **잠긴 표면(§5)**이라 차장님 직접 작업이 필요하고, **§5-4 Rule 3 선개정**과 **전역 파일 대조**가 선결 조건이다. 다음 단계: ① 전역 GEMINI.md 확보(붙여넣기/워크스페이스 복사) → ② 전역·로컬 중복 통합 검토(R1) → ③ 추출 가능/불가·잠김 구분한 최종 개정안 작성 → ④ 차장님 승인 후 안티 실행(잠긴 부분은 차장님 직접).
