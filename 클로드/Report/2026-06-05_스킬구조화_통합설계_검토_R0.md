## 1. 총평

`.agent/skills/`에 **65개**가 들어 있으나 QMS 워크플로에 실제로 배선된 커스텀 스킬은 **약 13개**뿐이고 나머지 ~50개는 외부에서 들여온 범용 라이브러리(anthropic superpowers, vercel-*, baoyu-*, 에이전트 설계용)다. 즉 **신호 13 : 잡음 50** 구조라 GEMINI.md가 호출하려는 스킬이 잡음에 묻힌다. 더 심각한 건 **규칙이 의존하는 스킬 두 개가 깨져 있다는 점** — `qms-orchestrator`는 SKILL.md가 아예 없고(스크립트 묶음, 그런데 §5-1 잠긴 표면), `plan-self-review`는 YAML 프런트매터가 없어(§5-5 필수 게이트인데) 자동 트리거가 안 된다. **규칙 구조화와 스킬 구조화는 한 몸**이라는 차장님 판단이 정확하다 — 규칙이 가리키는 스킬이 부실하면 위임이 무력화되기 때문이다. 방향: **① 깨진 핵심 스킬 복구 → ② 잡음 아카이브/정리 → ③ 중복 클러스터 통합 → ④ 규칙 §3/§4/§5 절차를 해당 스킬 본문으로 이전.**

4. **`plan-self-review` 본문에 §5-3 체크리스트 + §5-5 절차 흡수** — *§5 잠금 → 차장님.*

## 8. 주의사항

1. **삭제는 Human-controlled(§5-2).** 모든 제거는 하드 삭제가 아니라 **아카이브(이동/비활성)** 우선, 차장님 승인 후 `Remove-Item`.
2. **공유 라이브러리 가능성.** baoyu-*·vercel-*·superpowers는 다른 프로젝트와 공유/심링크일 수 있음 → 제거 전 출처·참조 확인(잘못 지우면 타 프로젝트 영향).
3. **잠긴 표면(§5·orchestrator 스크립트)은 차장님만.** 결함 1·2 복구는 안티에 지시 불가.
4. **전역 GEMINI.md 미확인.** 전역이 일부 스킬을 전제할 수 있음 → 전역 파일 받은 뒤 프루닝 확정.
5. **마운트 staleness.** 최종 삭제·이동 전 차장님 실터미널로 `.agent/skills` 실제 목록 교차확인.

## 9. 결론

65개 중 QMS 척추 13개를 살리고, 결함 2건(qms-orchestrator·plan-self-review)을 **최우선 복구**한 뒤, 스택무관·리버스API·에이전트설계 메타(~25개)를 아카이브/분리하고 중복 클러스터를 통합하면 신호:잡음 비가 정상화된다. 규칙 §3/§4/§5 절차는 §6 매핑대로 스킬 본문으로 이전한다. **선결: ① 전역 GEMINI.md 확보 ② §5-4 Rule3 개정(절차→SKILL 허용) ③ 삭제 승인.** 다음: 차장님 승인 범위 확정 → R1에서 아카이브 목록·통합 스펙·프런트매터 초안을 구체화.
