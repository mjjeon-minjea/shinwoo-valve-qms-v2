## 1. 프로젝트 개요

신우밸브주식회사 QMS v2 프로젝트(`shinwoo-valve-qms`)에 **하네스 엔지니어링**을 적용했다. 안티그래비티(Gemini AI)가 구현을 담당하고, 클로드가 각 단계를 검토·검증하는 역할 분담.

**하네스 엔지니어링 = AI 에이전트의 자율적 오작동을 제어하기 위한 4종 안전망 구조**
- Locked(잠긴 표면): 에이전트 수정 절대 금지
- Editable: 자유 수정 가능
- Append-only: 추가만 가능
- Human-controlled: 차장님 승인 후만 실행

## 6. 핵심 기술 규칙 (새 세션에서 반드시 숙지)

### QMS 기술 스택
- React (Vite SPA), JavaScript/JSX only — TypeScript 없음
- Supabase cloud (로컬 Docker 아님)
- PowerShell 환경 (rm 금지 → Remove-Item 사용)
- 환경변수: `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY` (`.env.local`)

### 영구 금지 규칙 (안티그래비티에게 적용)
- CLAUDE.md 생성 금지 — 전역 규칙은 GEMINI.md 전용
- `import.meta.env` Node.js에서 사용 금지 — dotenv + process.env 사용
- lazy RegEx `([^/'"]+?)` 금지 — greedy `([^/'"]+)` 사용
- `rm` 명령 금지 — `Remove-Item` 사용
- 잠긴 표면 4개 파일 수정 금지

### 배포 파이프라인 (3단 검증 필수)
- 테스트 배포: `test-origin` → `shinwoo-valve-qms.git` → `shinwoo-valve-qms.vercel.app`
- 실서버 배포: `origin` → `shinwoo-valve-qms-v2.git` → `shinwoo-valve-qms-v2.vercel.app`
- git push 전 반드시 차장님 승인 필요
