## 3. 최종 확인 필요 항목 — Windows에서 직접 실행 필요

클로드의 bash 샌드박스는 Linux 환경이라 Windows 프로젝트의 `npm` 명령을 직접 실행할 수 없다. **차장님이 아래 명령어를 직접 실행하여 최종 확인을 부탁드린다.**

```powershell
cd "C:\Users\mjjeon\Desktop\QMS 프로젝트\shinwoo-valve-qms"

# 1. JSON 유효성 최종 확인
node -e "require('./package.json'); console.log('JSON OK')"

# 2. 패키지 설치 (husky, lint-staged 등 신규 패키지)
npm install

# 3. 아키텍처 린트 통과 확인
npm run lint
```

### 예상 결과 안내

`npm run lint` 실행 시 아래 중 하나가 나올 수 있다:

**정상 완료 (아키텍처 이상 없음):**
```
✖ 19 problems (0 errors, 19 warnings)
```
단, `package.json`의 lint 스크립트에 `--max-warnings 0` 플래그가 있기 때문에 경고 19건만으로도 **exit code 1**로 종료될 수 있다. 이 경우 **아키텍처 boundaries 에러가 0건이면 하네스 목적은 달성된 것**이다. `no-unused-vars` 경고 19건은 하네스와 무관한 레거시 코드 문제다.

**만약 boundaries 에러가 발생한다면:** 에러 메시지를 클로드에게 공유하면 추가 분석을 드리겠다.
