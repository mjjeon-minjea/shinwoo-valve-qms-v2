## 1. 시스템 구조 현황

신우밸브 QMS는 3단계 환경으로 운영됩니다. Supabase 인스턴스는 각 환경별로 완전히 분리되어 있습니다.

| 단계 | 환경 | 용도 |
|------|------|------|
| 1단계 | 로컬 (Local) | 기능 개발 · 버그 수정 |
| 2단계 | 테스트웹 (Staging) | 실제 데이터 검증 |
| 3단계 | 메인웹 (Production) | 실제 운영 |

각 환경 Supabase가 분리되어 있으므로, 개발·테스트 단계 이슈가 운영 데이터에 직접 영향을 주지 않습니다.

## 3. 핵심 문제: 인수검사 화면이 왜 안 보이나?

Dashboard.jsx는 전체 대시보드를 담당하는 "메인 서랍장"입니다.
지금 이 서랍장 안에 인수검사 화면 코드도 같이 들어가 있습니다.
1,914줄 안에 인수검사 코드가 섞여있어서, 인수검사 에이전트가 "내 파일"을 찾을 수 없는 상태입니다.

**해결 방법: 인수검사 코드를 꺼내서 별도 파일로 만들기**

- 꺼내기 전: Dashboard.jsx 열면 1,914줄 전체를 봐야 함
- 꺼낸 후: InboundAnalysis.jsx, InboundHistory.jsx 각자의 파일에서 작업 가능
- 효과: 인수검사 고쳐도 주간보고·공정검사 건드릴 일 없음

지금 인수검사 고치면 주간업무가 고장나는 이유가 바로 이것입니다. 코드가 한 파일에 뭉쳐있으니 어디를 수정하든 영향이 퍼집니다.

## 4. 에이전트 구조 현황 (생성 완료)

.agent/_agents/ 폴더에 6개 에이전트 생성 완료.
각 에이전트는 scope.md에 담당 .jsx 파일 목록이 명시되어 있습니다.

| 에이전트 | 담당 DB 테이블 | 상태 |
|---------|--------------|------|
| 게시판 | notices, resources, dev_notes, suggestions | 준비 완료 |
| 인수검사 | inspections, item_master | 추출 선행 필요 |
| 공정검사 | process_inspections | 즉시 사용 가능 |
| 최종검사 | final_inspections (설계 예정) | 설계 대기 |
| 업무관리 | weekly_reports | 준비 완료 |
| 관리자 | users, settings | 준비 완료 |

**각 에이전트 담당 파일 목록**

- 게시판: NoticeBoard, ResourceRoom, DevNotes, PostApproval, Suggestions
- 인수검사: InspectionAnalysisDashboard, NonConformanceStatus + InboundAnalysis(추출필요), InboundHistory(추출필요)
- 공정검사: ProcessInspectionDashboard, ProcessHistory, ProcessAnalysis, WorkplaceAnalysis, EquipmentAnalysis, ModelCategoryAnalysis
- 최종검사: (미구현)
- 업무관리: WeeklyReport, WeeklyStatus, CalendarView, Chatbot
- 관리자: UserManagement, PasswordChangeModal, Header, Hero

**에이전트 폴더 구조 (4개 파일)**

```
_agents/[에이전트명]/
├── prompt.md  — 역할 정의, 작업 규칙
├── scope.md   — 담당 파일 목록 + 수정 금지 구역
├── memory.md  — 버그 교훈, 주의사항 누적
└── status.md  — 현재 미완성 기능, 알려진 버그
```
