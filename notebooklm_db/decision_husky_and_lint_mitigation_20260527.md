# [의사결정] QMS v2 Husky 및 Lint 제약 사항 완화 결정 (2026-05-27)

* **의사결정 ID:** `decision_husky_and_lint_mitigation_20260527`
* **의사결정 일자:** 2026-05-27
* **기록 대상:** 신우밸브 QMS v2 프로젝트 하네스 엔지니어링 2단계 (아키텍처 제약)

---

## 1. 배경 및 맥락
하네스 아키텍처 제약을 수립하기 위해 `husky` 및 `eslint-plugin-boundaries`를 도입하였으나, 개발 서버나 Windows 로컬 환경에서 다음과 같은 빌드 중단 장애가 감지되었습니다:
1. `npm run lint` 실행 시, 기존 레거시 코드에 포함되어 있던 19개의 미사용 변수 경고(`no-unused-vars`) 등으로 인해 `--max-warnings 0` 제약에 가로막혀 exit code `1`로 빌드가 실패함.
2. `husky`를 통한 강제 아키텍처 boundaries 제약이 로컬 기동 시 일부 플랫폼이나 특정 git 환경에서 인터프리터 오류를 발생시키며 commit 진행을 완전히 차단하는 현상이 관측됨.

## 2. 채택된 기술적 결정
전민재 차장님의 승인 하에 아래 두 가지 package.json 스크립트 수정안을 최종 반영하기로 결정하였습니다:
* **수정 1 (prepare):** `"prepare": "husky"` ➔ `"prepare": "husky || true"`
  - **결정 사유:** husky 실행이 불가한 예외적인 환경에서도 빌드와 커밋 라이프사이클이 멈추지 않고 안전하게 우회 기동되도록 안정장치 격리망 마련.
* **수정 2 (lint):** `eslint . ... --max-warnings 0` ➔ `eslint . ... --max-warnings 30`
  - **결정 사유:** 기존 레거시 파일들의 미사용 경고(현재 19건)가 아키텍처 boundaries 에러가 아님에도 빌드를 완전 중단시키는 현상을 방지하기 위해, 최대 경고 한도를 `30`으로 유연하게 완화하여 정식 빌드 및 린트 검사 통과를 보장함.

## 3. 결과 및 트레이드오프
* **장점:** Boundaries 단방향 아키텍처 규칙 위반 에러(`0 errors`)인 클린 빌드 상태를 달성함과 동시에 불필요한 빌드 마비 방지.
* **트레이드오프:** 최대 경고 한도를 30으로 느슨하게 두었으므로 경고 누적을 수동 모니터링해야 하는 리스크가 있으나, boundaries 규칙 자체의 무결성(0 errors)은 그대로 유지되므로 아키텍처 정합성을 지키는데 무리 없음.
