# 📋 [의사결정] AI 기억 유실 파훼를 위한 경량 메모리 프로토콜(Lean Memory) 채택

**날짜**: 2026-04-21
**참석자**: 전민재 차장(신우밸브 QA), Antigravity(AI 수석 개발자)
**카테고리**: 의사결정 (Decision)

---

## 1. 이슈 개요 (Context)
- AI 모델의 세션 전환 시 발생하는 '기억상실(Context Loss)' 현상으로 인한 업무 연속성 저해 및 중복 보고의 비효율성 해결 필요.
- 기존에 제안된 비대한 JSON 기반 동기화 및 복잡한 규칙 주입 방식의 문제점 노출.

## 2. 검토 및 반박 (Counter-Arguments)
차장님의 기술적 통찰에 의해 아래의 기존 안들이 기각됨:
1.  **JSON 데이터화**: 파일이 커질수록 토큰 낭비가 심해지며 시스템 부하 증가.
2.  **규칙(Rules) 비대화**: `AI_INTERACTION_RULES.md`에 조건이 많아질수록 AI의 추론 능력이 저하(Stupidity)되는 현상 발생.
3.  **외부 API 의존**: NotebookLM API 등의 불안정성으로 인한 시스템 신뢰도 저하.

## 3. 최종 결정 (Decision)
토큰 연산 효율과 로컬 안정성을 최우선으로 하는 **'경량 메모리 프로토콜(Lean Memory Protocol)'**을 정식 채택함.

### 🛠️ 핵심 실행 지침 (Execution SOP)
- **[Pointer]**: 프로젝트 루트에 100바이트 미만의 초소형 상태 파일(`.todo`)을 운용하여 마지막 작업 위치만 마킹함.
- **[Local Context]**: 전역 규칙 대신, 현재 작업 중인 소스 코드 상단에 `// TODO` 주석을 활용하여 컨텍스트를 유지함.
- **[Archive]**: 세션 종료 시 `README.md` 하단에 3줄 내외의 작업 요약을 문자로 각인하여 물리적 영속성을 확보함.

## 4. 기대 효과
- **비용 절감**: 불필요한 토큰 소비 최소화.
- **지능 유지**: 룰셋 다이어트를 통한 AI의 명확한 판단력 유지.
- **무손실 기억**: 로컬 파일 시스템 기반의 확실한 작업 연속성 확보.
